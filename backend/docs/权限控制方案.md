## 一、核心原则

| 原则 | 说明 |
|------|------|
| **作业区归属** | 作业区只属于甲方公司（`work_areas.company_id` → 甲方公司） |
| **作业区管理权** | 仅**系统管理员** + **甲方管理员**可创建/删除/修改作业区 |
| **乙方/监理访问权** | 只能查看/操作**负责的作业区**（通过 `user_work_areas` 绑定） |
| **用户管理权** | 各公司管理员管理**本公司**用户 |
| **个人中心** | **所有用户**都有个人中心权限 |

## 二、权限模型架构

```
┌─────────────────────────────────────────────────────────────────┐
│                        权限控制架构                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   用户 (User)                                                   │
│     │                                                           │
│     ├──→ 角色 (Role) ──→ 角色权限 (Role_Permission) ──→ API 权限  │
│     │         │                                                │
│     │         └──→ 角色菜单权限 (Role_Menu_Permission)          │
│     │                                                           │
│     └──→ 用户自定义权限 (User_Permission) [覆盖角色默认]         │
│                                                                 │
│   数据权限 (ABAC)                                                │
│     │                                                           │
│     └──→ 用户作业区 (User_Work_Area) ──→ 数据过滤规则            │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 设计原则

1. **API 即权限**：每个 API 端点对应一个权限代码
2. **前端元素绑定 API**：按钮/操作对应到具体 API 权限
3. **角色默认授权**：角色预置一组 API 权限
4. **用户可覆盖**：可为单个用户增删权限（优先级高于角色）
5. **菜单权限独立**：菜单可见性由菜单权限控制
6. **数据权限补充**：甲方作业区用户只能访问绑定的作业区数据

## 三、数据库表结构

### 3.1 现有表（保持不变）

| 表名 | 用途 |
|------|------|
| `users` | 用户表 |
| `roles` | 角色表（22 个角色） |
| `permissions` | 权限表（47 条权限） |
| `user_roles` | 用户 - 角色关联 |
| `role_permissions` | 角色 - 权限关联 |
| `menus` | 菜单表（6 个菜单） |
| `role_menu_permissions` | 角色 - 菜单权限关联 |
| `user_menu_permissions` | 用户 - 菜单权限关联（覆盖用） |
| `user_work_areas` | 用户 - 作业区关联（数据权限） |

### 3.2 新增表 1：API 权限映射表

```sql
CREATE TABLE api_permissions (
    id              BIGSERIAL PRIMARY KEY,
    api_path        VARCHAR(255) NOT NULL,      -- API 路径，如 /user/list
    api_method      VARCHAR(20)  NOT NULL,      -- HTTP 方法，如 GET/POST
    permission_id   INTEGER      NOT NULL,      -- 关联 permissions 表
    description     VARCHAR(500),               -- 描述
    is_public       BOOLEAN      DEFAULT false, -- 是否公开（无需权限）
    created_at      TIMESTAMP    DEFAULT CURRENT_TIMESTAMP,
    updated_at      TIMESTAMP    DEFAULT CURRENT_TIMESTAMP,
    UNIQUE (api_path, api_method)
);
```

### 3.3 新增表 2：UI 元素权限映射表

```sql
CREATE TABLE ui_element_permissions (
    id              BIGSERIAL PRIMARY KEY,
    page_code       VARCHAR(50)  NOT NULL,      -- 页面编码
    element_type    VARCHAR(20)  NOT NULL,      -- 元素类型：button/select/menu/item
    element_id      VARCHAR(100) NOT NULL,      -- 元素 ID/标识
    element_name    VARCHAR(200) NOT NULL,      -- 元素名称
    permission_id   INTEGER      NOT NULL,      -- 关联 permissions 表
    action_type     VARCHAR(20)  DEFAULT 'write', -- 操作类型：read/write/delete
    visible_default BOOLEAN      DEFAULT true,  -- 默认是否可见
    enabled_default BOOLEAN      DEFAULT true,  -- 默认是否可操作
    created_at      TIMESTAMP    DEFAULT CURRENT_TIMESTAMP,
    UNIQUE (page_code, element_type, element_id)
);
```

| 监理工程师 | 负责作业区 | `LEFT JOIN user_work_areas uwa ON wa.id = uwa.work_area_id WHERE uwa.user_id = :userId` |

### 7.3 数据权限实现示例（Java）

```java
public List<WorkArea> getAccessibleWorkAreas(User user) {
    String sql = "SELECT wa.* FROM work_areas wa";
    Map<String, Object> params = new HashMap<>();
    
    // 系统管理员：不过滤
    if (user.hasRole("ROLE_SYSTEM_ADMIN")) {
        return jdbcTemplate.query(sql, params, workAreaRowMapper);
    }
    
    // 甲方角色
    if (user.getRoleCode().startsWith("ROLE_JIAFANG")) {
        // 甲方管理员/项目经理：按公司过滤
        if (user.getRoleCode().equals("ROLE_JIAFANG_ADMIN") ||
            user.getRoleCode().equals("ROLE_JIAFANG_PM")) {
            sql += " WHERE wa.company_id = :companyId";
            params.put("companyId", user.getCompanyId());
        }
        // 甲方作业区角色：按 user_work_areas 过滤
        else if (user.getRoleCode().contains("WORK_AREA")) {
            sql += " JOIN user_work_areas uwa ON wa.id = uwa.work_area_id";
            sql += " WHERE uwa.user_id = :userId";
            params.put("userId", user.getId());
        }
    }
    
    // 乙方角色：按 user_work_areas 过滤
    else if (user.getRoleCode().startsWith("ROLE_YIFANG")) {
        sql += " JOIN user_work_areas uwa ON wa.id = uwa.work_area_id";
        sql += " WHERE uwa.user_id = :userId";
        params.put("userId", user.getId());
    }
    
    // 监理方角色
    else if (user.getRoleCode().startsWith("ROLE_JIANLIFANG")) {
        // 监理管理员：全部可见
        if (user.getRoleCode().equals("ROLE_JIANLIFANG_ADMIN")) {
            // 不过滤
        }
        // 监理工程师/项目经理：按 user_work_areas 过滤
        else {
            sql += " LEFT JOIN user_work_areas uwa ON wa.id = uwa.work_area_id";
            sql += " WHERE uwa.user_id = :userId";
            params.put("userId", user.getId());
        }
    }
    
    return jdbcTemplate.query(sql, params, workAreaRowMapper);
}
```

### 8.4 权限拦截器

```java
@Component
public class PermissionInterceptor implements HandlerInterceptor {
    
    @Autowired
    private PermissionService permissionService;
    
    @Override
    public boolean preHandle(HttpServletRequest request, 
                            HttpServletResponse response, 
                            Object handler) {
        User user = SecurityContextHolder.getCurrentUser();
        String apiPath = request.getRequestURI();
        String method = request.getMethod();
        
        boolean hasPermission = permissionService.hasApiPermission(
            user, apiPath, method);
        
        if (!hasPermission) {
            response.setStatus(403);
            response.getWriter().write("无权限访问");
            return false;
        }
        
        return true;
    }
}
```

```javascript
router.beforeEach((to, from, next) => {
  const userStore = useUserStore();
  const menus = userStore.menus || [];
  const menuCode = to.meta.menuCode;
  
  // 个人中心始终可访问
  if (menuCode === 'profile') {
    next();
    return;
  }
  
  // 检查菜单权限
  const hasPermission = menus.some(m => m.menuCode === menuCode);
  
  if (!hasPermission) {
    next({ name: 'Home' });
    return;
  }
  
  next();
});
```
