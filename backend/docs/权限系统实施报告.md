## 四、权限模型

### 4.1 权限计算逻辑

```
用户最终权限 = (所有角色权限的并集) ∪ (用户 GRANT 权限) - (用户 DENY 权限)

特殊规则:
- 系统管理员拥有所有权限 (*:*:*)
- 数据权限通过 @DataScope 注解自动注入
```

### 4.2 权限分层

```
L1: 功能权限 - 能否访问 API
L2: 数据权限 - 能访问哪些数据
L3: 字段权限 - 预留
```

## 五、使用指南

### 5.1 系统管理员操作流程

#### 配置角色权限

1. 登录系统（使用 system_admin 角色）
2. 进入"系统管理" → "角色权限配置"
3. 选择要配置的角色
4. 勾选/取消勾选权限
5. 填写配置备注（用于审计日志）
6. 点击"保存配置"

#### 查看审计日志

1. 进入"系统管理" → "审计日志"
2. 使用筛选条件（操作类型、目标类型、操作人）
3. 点击"详情"查看完整的变更内容

### 5.2 开发人员使用指南

#### 后端添加权限控制

```java
// 1. 在 Controller 方法上添加注解
@ApiPermission("user:create")
@PostMapping("/create")
public Result<User> createUser(@RequestBody UserCreateDTO dto) {
    // ...
}

// 2. 添加数据权限
@DataScope(tableAlias = "w", scopeType = ScopeType.WORK_AREA)
@GetMapping("/list")
public Result<List<WorkArea>> listWorkAreas() {
    // ...
}
```

#### 前端添加权限控制

```vue
<!-- 1. 使用指令 -->
<el-button v-permission="'user:create'">创建用户</el-button>

<!-- 2. 使用组件 -->
<PermButton permission="user:delete">删除用户</PermButton>

<!-- 3. 在 JS 中检查 -->
<script setup>
const userStore = useUserStore()
if (userStore.permissions.includes('user:create')) {
  // 有权限
}
</script>
```
