# 用户管理功能实现总结

## 实现时间
2026-03-11

## 功能概述

实现了完整的用户管理体系，支持三种注册方式和分级审批流程。

## 已实现的功能

### 1. 三种注册方式

#### 1.1 用户自主注册（需审批）
- **文件**: `RegisterRequest.java`, `UserService.register()`
- **接口**: `POST /user/register`
- **特点**: 
  - 用户填写基本信息提交注册
  - 自动设置为"待审批"状态
  - 审批通过前无法登录
  - 支持用户名、邮箱、手机号唯一性校验

#### 1.2 管理员手动创建
- **文件**: `CreateUserRequest.java`, `UserService.createUser()`
- **接口**: `POST /user`
- **特点**:
  - 管理员可创建用户并分配角色
  - 支持 `autoApprove` 参数控制是否直接通过审批
  - 权限控制：管理员只能创建自己公司类型的用户
  - 系统管理员可创建所有用户

#### 1.3 管理员批量导入
- **文件**: `BatchImportRequest.java`, `UserService.batchImport()`
- **接口**: `POST /user/batch-import`
- **特点**:
  - 支持批量导入多个用户
  - 返回导入结果（总数、成功数、失败数、错误信息）
  - 支持部分成功（失败的用户不影响其他用户）
  - 支持 `autoApprove` 参数

### 2. 审批流程

#### 2.1 用户审批
- **文件**: `ApprovalRequest.java`, `UserService.approveUser()`
- **接口**: `POST /user/approve`
- **特点**:
  - 支持通过/拒绝两种操作
  - 拒绝时必须填写拒绝原因
  - 防止重复审批
  - 权限控制：管理员只能审批自己公司类型的用户

#### 2.2 待审批用户列表
- **接口**: `GET /user/pending-approvals`
- **特点**:
  - 根据审批人权限过滤待审批用户
  - 系统管理员可看到所有待审批用户
  - 其他管理员只能看到本公司类型的待审批用户

### 3. 查询功能

#### 3.1 分页查询用户列表
- **文件**: `UserQueryRequest.java`, `UserService.queryUsers()`
- **接口**: `GET /user/list`
- **支持筛选**:
  - 用户名（模糊）
  - 真实姓名（模糊）
  - 公司 ID
  - 公司类型 ID
  - 角色 ID
  - 审批状态
  - 用户状态
  - 关键词（综合搜索）

#### 3.2 用户详情
- **接口**: `GET /user/{id}`
- **返回**: 完整用户信息（包括公司、角色、审批状态等）

### 4. 权限控制

#### 4.1 创建用户权限
- **系统管理员** (company_type_id=4): 可创建所有公司类型的用户
- **甲方管理员** (company_type_id=1): 只能创建甲方用户
- **乙方管理员** (company_type_id=2): 只能创建乙方用户
- **监理方管理员** (company_type_id=3): 只能创建监理方用户

#### 4.2 审批用户权限
- **系统管理员**: 可审批所有用户
- **其他管理员**: 只能审批同公司类型的用户

#### 4.3 实现方式
- `UserService.validateUserCreationPermission()`: 校验创建权限
- `UserService.validateApprovalPermission()`: 校验审批权限
- 在业务逻辑层进行校验，确保安全性

## 数据库字段

### users 表（已有字段）
- `approval_status`: 审批状态（0:待审批 1:通过 2:拒绝）
- `approved_by`: 审批人 ID
- `approved_at`: 审批时间
- `rejection_reason`: 拒绝原因
- `status`: 用户状态（0:待审批 1:正常 2:禁用）

## 错误码

- `USER_NOT_APPROVED(2004)`: 用户尚未审批通过
- `USERNAME_EXISTS(2005)`: 用户名已存在
- `EMAIL_EXISTS(2006)`: 邮箱已被注册
- `PHONE_EXISTS(2007)`: 手机号已被注册
- `FORBIDDEN(403)`: 无权限访问

## 安全考虑

1. **密码加密**: 所有密码使用 BCrypt 加密存储
2. **事务控制**: 创建用户和分配角色在同一事务中
3. **权限校验**: 在业务层进行权限校验，防止越权操作
4. **唯一性校验**: 用户名、邮箱、手机号全局唯一
5. **审批流程**: 自主注册用户必须经过审批才能登录
