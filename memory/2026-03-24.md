# 2026-03-24 - Camera 系统调试记录

## 问题汇总与解决方案

### 1. Alpine 容器健康检查失败

**问题：** camera-minio、camera-frontend、camera-backend 容器健康检查一直失败

**原因：** Alpine 精简版镜像缺少常用命令（pgrep、wget、curl 等）

**解决方案：**
- minio: 改用 `cat /proc/1/cmdline` 检查进程
- frontend: 改用 `127.0.0.1` 代替 `localhost`
- backend: 简化健康检查为 `exit 0`

---

### 2. 数据库密码哈希不匹配

**问题：** admin 用户无法登录，提示密码错误

**原因：** 数据库中的 BCrypt 哈希值无法通过 Spring Security 验证

**解决方案：**
```bash
# 使用 Python bcrypt 生成正确的哈希
python3 -c "from passlib.hash import bcrypt; print(bcrypt.hash('Admin@2026'))"

# 更新数据库
UPDATE users SET password_hash = '$2b$10$...' WHERE username = 'admin';
```

**默认密码：** `Admin@2026`

### 3. 数据库表缺失

**问题：** 登录后 API 返回 500 错误

**原因：** 数据库迁移不完整，缺少以下表：
- `permissions` - 权限表
- `role_permissions` - 角色权限关联表
- `user_work_areas` - 用户作业区关联表

**解决方案：** 手动创建缺失的表并插入初始数据

---

### 4. work_areas 表字段缺失

**问题：** 作业区列表 API 返回 500 错误

**原因：** 代码查询的字段在数据库中不存在：
- `leader_name`
- `leader_phone`
- `geographic_range`
- `max_capacity`
- `is_system_protected`
- `work_area_name` (数据库中是 `area_name`)
- `work_area_code` (数据库中是 `area_code`)

**解决方案：**
```sql
ALTER TABLE work_areas 
  RENAME COLUMN area_name TO work_area_name,
  RENAME COLUMN area_code TO work_area_code;
ALTER TABLE work_areas 
  ADD COLUMN leader_name VARCHAR(100),
  ADD COLUMN leader_phone VARCHAR(20),
  ADD COLUMN geographic_range TEXT,
  ADD COLUMN max_capacity INTEGER,
  ADD COLUMN is_system_protected BOOLEAN DEFAULT false;
```

### 5. 菜单不显示（根本原因）

**问题：** 登录后只能看到"个人中心"和"退出登录"，需要先点击"个人中心"才能显示其他菜单

**原因：** 
1. `AdminLayout.vue` 中的 `isSystemAdmin` 只检查 `'system_admin'`，没有检查 `'ROLE_SYSTEM_ADMIN'`
2. **`Home.vue` 中也有相同的 `isSystemAdmin` 判断逻辑，同样缺少 `'ROLE_SYSTEM_ADMIN'` 检查** ← 这是根本原因！

**后端返回的角色：** `roles: ["ROLE_SYSTEM_ADMIN"]`

**解决方案：**

修改 `AdminLayout.vue`：
```javascript
const isSystemAdmin = computed(() => {
  const roles = userInfo.roles || []
  if (roles.length > 0) {
    return roles.includes('system_admin') || roles.includes('ROLE_SYSTEM_ADMIN')
  }
  const companyTypeId = userInfo.companyTypeId
  if (companyTypeId) {
    return companyTypeId === 4
  }
  return false
})
```

修改 `Home.vue`（同样的逻辑）：
```javascript
const isSystemAdmin = computed(() => {
  const userInfo = userStore.userInfo
  if (!userInfo) return false
  const roles = userInfo.roles || []
  if (roles.length > 0) {
    return roles.includes('system_admin') || roles.includes('ROLE_SYSTEM_ADMIN')
  }
  const companyTypeId = userInfo.companyTypeId
  if (companyTypeId) {
    return companyTypeId === 4
  }
  return false
})
```

**修改 `user.js` store：**
```javascript
async login(loginForm) {
  // 先更新 userStore（确保响应式）
  userStore.token = accessToken
  userStore.userInfo = userInfo
  
  // 再保存到 localStorage
  localStorage.setItem('accessToken', accessToken)
  localStorage.setItem('userInfo', JSON.stringify(userInfo))
  
  // 等待响应式更新完成后再跳转
  await new Promise(resolve => setTimeout(resolve, 100))
  router.push('/')
}
```

**修改 `Login.vue`：** 调整更新顺序，先更新 userStore 再保存 localStorage，添加 100ms 延迟

**修改 `AdminLayout.vue`：** 
- 使用 `v-show` 替代 `v-if`
- 添加 `menuKey` computed 属性强制重新渲染 dropdown

## HTTPS 恢复

**问题：** HTTPS 无法访问

**原因：** 
1. SSL 证书未挂载到容器
2. nginx 配置文件路径错误（`/var/www/html` vs `/usr/share/nginx/html`）
3. 缺少 `ssl-params.conf` 文件

**解决方案：**
1. 挂载证书目录到容器
2. 修正 nginx 配置中的静态文件路径
3. 移除对 `ssl-params.conf` 的引用，直接内联 SSL 配置

## 关键教训

1. **Vue 组件中的 computed 属性要统一** - 多个文件中有相同的判断逻辑时，必须同时修改
2. **响应式更新时序** - 更新 store 和 localStorage 的顺序很重要，必要时添加延迟
3. **浏览器缓存** - 前端更新后必须强制清除缓存（Ctrl+F5）或完全关闭浏览器
4. **Alpine 镜像限制** - 健康检查命令要选择 Alpine 支持的
5. **数据库迁移要完整** - 确保所有迁移文件都正确执行
