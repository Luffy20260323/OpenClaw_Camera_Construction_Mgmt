## 🔧 Docker 构建优化（重要）

### 问题
- Docker 构建时每次都要下载 Maven/npm 依赖
- 构建时间长（5-15 分钟），网络依赖严重

### 解决方案
采用"宿主机编译 + Docker 复制"方式：
- 宿主机编译（利用本地缓存依赖）
- Docker 只复制编译产物（秒级构建）

### 修改的文件
- `backend/Dockerfile` - 改为直接复制 jar
- `frontend/Dockerfile` - 改为直接复制 dist
- `backend/build-docker.sh` - 新增构建脚本
- `frontend/build-docker.sh` - 新增构建脚本
- `BUILD.md` - 新增完整构建指南

### 标准流程
```bash
# 后端
cd backend && mvn clean package -DskipTests && docker build -t camera-backend .

# 前端
cd frontend && npm run build && docker build -t camera-frontend .
```

### 性能提升
- 后端构建：10 分钟 → 1-2 分钟（提升 80%）
- 前端构建：5 分钟 → 30 秒（提升 90%）
- 镜像体积：减小 700MB

### 注意事项
- ⚠️ 必须先编译再构建 Docker
- 浏览器缓存需强制刷新 (Ctrl+Shift+R)

## 📋 会话记忆问题

### 问题描述
用户发现会话中断超过 4 小时后，之前的对话会被忘记。

### 原因分析
1. OpenClaw 为每次新对话创建新的会话文件
2. 新会话不会自动加载旧会话的历史
3. 只有 `MEMORY.md` 和 `memory/YYYY-MM-DD.md` 会加载

### 解决方案
1. **重要信息写入 MEMORY.md** - 长期记忆，所有会话共享
2. **每日日志记录** - `memory/YYYY-MM-DD.md` 自动加载
3. **项目文档** - `BUILD.md` 等文件永久保存

### 记忆系统说明
| 类型 | 持久性 | 加载时机 |
|------|--------|---------|
| 会话历史 | 临时 | 仅当前会话 |
| MEMORY.md | 永久 | 主会话启动时 |
| memory/*.md | 永久 | 所有会话启动时 |
| 项目文档 | 永久 | 需要时读取 |
