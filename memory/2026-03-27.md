# 2026-03-27 - 管理页面对话框问题修复与定位教训

## 🐛 Bug #001: 管理页面创建/编辑对话框无法弹出

### 问题描述
- **影响页面**: 公司管理、角色管理、作业区管理
- **现象**: 点击"新建"或"编辑"按钮后对话框不弹出
- **报告时间**: 2026-03-26 07:24
- **解决时间**: 2026-03-26 18:50

### 根本原因
1. **代码层面**: 三个 Vue 组件缺少 `<el-dialog>` 组件定义
   - `CompanyList.vue`
   - `RoleList.vue`
   - `WorkAreaList.vue`

2. **部署层面** (主要延误原因): 
   - 前端使用 Docker 容器部署 (`camera-frontend`)
   - 编译后的文件未部署到容器内
   - 宿主机 `/var/www/html` 的修改不影响容器

### 排查过程（含错误尝试）

#### ❌ 第一次尝试 (15 分钟)
- 假设：代码逻辑问题
- 行动：修改代码 → 编译 → 部署到 `/var/www/html`
- 结果：失败（部署目标错误）

#### ❌ 第二次尝试 (10 分钟)
- 假设：浏览器缓存问题
- 行动：指导用户清除缓存
- 结果：失败（容器内文件未更新）

#### ✅ 第三次尝试 (5 分钟)
- 行动：`docker ps` 确认部署架构
- 发现：`camera-frontend` 容器
- 正确部署：`docker cp dist/. camera-frontend:/usr/share/nginx/html/ && docker restart camera-frontend`
- 结果：成功

### 核心教训

> **如果一个问题反复解决不了，80% 是因为定位思路错了，不是问题本身难。**

**错误假设**:
- "编译了就应该生效"
- "部署到 `/var/www/html` 就可以了"

**正确做法**:
- 先 `docker ps` 确认部署方式
- 不要假设，要验证
- 同一个问题尝试 2 次没解决 → 立即触发系统性检查
