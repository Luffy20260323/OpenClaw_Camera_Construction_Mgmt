## 使用指南

### 查看文档

1. 登录系统
2. 进入 **系统管理** → **文档中心**
3. 找到 **"权限设计方案 V4.3"**（有🏆推荐标记）
4. 点击 **"查看"** 按钮
5. 在弹出的对话框中在线阅读

### 下载文档

1. 在文档卡片上点击 **"下载"** 按钮
2. 或打开文档后点击对话框中的下载按钮

### 搜索文档

在搜索框输入关键词：
- "权限" - 搜索所有权限相关文档
- "V4.3" - 搜索 V4.3 版本文档
- "审计" - 搜索包含审计内容的文档

## 文档内容概览

### V4.3 核心变更

| 方面 | V4.1 | V4.3 | 改进 |
|------|------|------|------|
| **表命名** | permission 表 | resource 表 | 语义清晰 |
| **权限体现** | 混淆在表名中 | permission 表 | 关系明确 |
| **数据范围** | 按资源配置 | 按模块配置 | 简化设计 |
| **审计日志** | 无 | 完整审计日志 | 可追溯 |
| **软删除** | 无 | 级联软删除 | 支持恢复 |

### 主要章节

1. **核心理念** - 资源与权限分离设计
2. **权限驱动开发流程** - 8 步完整流程
3. **资源与权限模型** - 核心概念和层级结构
4. **权限码命名规范** - module:resource:action:type
5. **数据库表结构** - resource、permission 等表设计
6. **两种管理员角色** - 超级管理员 vs 系统管理员
7. **数据范围权限** - ALL/COMPANY/DEPT/SELF/CUSTOM
8. **权限继承与冲突** - 不继承原则和优先级规则
9. **后端实现** - 注解、拦截器、缓存
10. **前端实现** - 指令、路由守卫、刷新菜单
11. **权限缓存** - Redis 缓存策略
12. **审计日志** - 完整的审计追踪
13. **实施步骤** - 5 个阶段 7 天计划
