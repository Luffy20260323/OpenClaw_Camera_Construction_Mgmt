## 一、资源管理模块（3 个页面）

### 1.1 资源列表页面 (`/system/resource`)

**功能描述**：展示所有资源的树形结构，支持增删改查

**功能点**：

| 序号 | 功能 | 说明 | 优先级 |
|------|------|------|--------|
| 1.1.1 | 树形表格展示 | 展示资源的层级结构（MODULE → MENU → PAGE → ELEMENT → API） | P0 |
| 1.1.2 | 资源类型筛选 | 按类型筛选：MODULE/MENU/PAGE/ELEMENT/API | P0 |
| 1.1.3 | 模块筛选 | 按模块编码筛选：system/user/role/part 等 | P0 |
| 1.1.4 | 搜索功能 | 按资源名称/编码/权限码模糊搜索 | P0 |
| 1.1.5 | 新建资源 | 创建新资源（支持所有类型） | P0 |
| 1.1.6 | 编辑资源 | 修改资源名称、排序、可见性等 | P0 |
| 1.1.7 | 删除资源 | 软删除资源（级联删除子资源） | P0 |
| 1.1.8 | 设置父资源 | 修改资源的父资源（带循环引用检查） | P0 |
| 1.1.9 | 批量设置父资源 | 批量修改多个资源的父资源 | P1 |
| 1.1.10 | 权限码自动生成 | 根据资源名称和模块自动生成权限码 | P1 |
| 1.1.11 | 资源路径预览 | 显示资源的完整父级路径 | P1 |
| 1.1.12 | 导出资源列表 | 导出为 Excel/JSON | P2 |

**界面元素**：
- 树形表格（el-table with tree-props）
- 筛选栏（类型下拉、模块下拉、搜索框）
- 新建/编辑/删除按钮
- 设置父资源对话框
- 权限码自动生成提示

**API 调用**：
- `GET /api/resource/tree` - 获取资源树
- `GET /api/resource` - 获取资源列表（分页）
- `POST /api/resource` - 创建资源
- `PUT /api/resource/{id}` - 更新资源
- `DELETE /api/resource/{id}` - 删除资源（软删除）
- `PUT /api/resource/{id}/parent` - 设置父资源
- `PUT /api/resource/batch-parent` - 批量设置父资源
- `POST /api/resource/check-cycle` - 检查循环引用
- `GET /api/resource/{id}/path` - 获取资源路径
