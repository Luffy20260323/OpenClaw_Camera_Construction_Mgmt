## 四、HTTPS 配置指南（参考）

### 4.1 为什么需要 HTTPS

当前系统使用 HTTP 协议，浏览器会提示"不安全"。HTTPS 的作用：
- 🔒 加密传输数据，防止窃听
- ✅ 消除浏览器"不安全"警告
- 🛡️ 防止中间人攻击
- 📋 满足合规要求

### 4.2 获取 SSL 证书

#### 方案一：Let's Encrypt 免费证书（推荐）✅ 已采用

```bash
# 1. 安装 Certbot
sudo apt update
sudo apt install certbot python3-certbot-nginx

# 2. 获取证书（确保域名已解析到服务器 1.95.153.137）
sudo certbot --nginx -d camera1001.pipecos.cn

# 3. 自动续期（Certbot 会自动配置定时任务）
sudo certbot renew --dry-run
```

#### 方案二：阿里云/腾讯云免费证书

1. 登录云服务商控制台
2. 申请免费 SSL 证书（通常有效期 1 年）
3. 下载 Nginx 格式的证书文件（.crt 和 .key）
4. 上传到服务器：`/etc/nginx/ssl/`

### 3.3 Nginx HTTPS 配置

编辑 Nginx 配置文件（`/etc/nginx/sites-available/camera` 或 `/etc/nginx/nginx.conf`）：

```nginx
# HTTP 重定向到 HTTPS
server {
    listen 80;
    server_name camera1001.pipecos.cn 1.95.153.137;
    return 301 https://$server_name$request_uri;
}

# HTTPS 配置
server {
    listen 443 ssl http2;
    server_name camera1001.pipecos.cn 1.95.153.137;

    # SSL 证书路径（根据实际获取方式调整）
    ssl_certificate /etc/letsencrypt/live/camera1001.pipecos.cn/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/camera1001.pipecos.cn/privkey.pem;

    # SSL 优化配置
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;
    ssl_session_cache shared:SSL:10m;
    ssl_session_timeout 10m;

    # 前端静态文件
    location / {
        root /root/.openclaw/workspace/OpenClaw_Camera_Construction_Mgmt/frontend/dist;
        try_files $uri $uri/ /index.html;
    }

    # 后端 API 反向代理
    location /api {
        proxy_pass http://localhost:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

### 3.4 重启 Nginx

```bash
# 测试配置是否正确
sudo nginx -t

# 重启 Nginx
sudo systemctl restart nginx
```

### 3.5 后端配置调整

更新 `application.yml` 或 `application-prod.yml`：

```yaml
# JWT 配置（建议开启 HTTPS 强制）
```

## 四、安全要求

### 4.1 鉴权要求

**重要**: 所有可能通过 HTTP/HTTPS 访问的资源，从公网访问时**必须进行鉴权**，**禁止非授权访问**。

| 资源类型 | 鉴权要求 | 说明 |
|----------|----------|------|
| API 接口 | ✅ 必须鉴权 | 除登录、注册、验证码等公开接口外，所有 API 需要 JWT Token |
| Swagger 文档 | ✅ 必须鉴权 | 禁止从公网直接访问 `/api/doc.html` |
| 前端页面 | ✅ 必须鉴权 | 所有页面需要登录后访问 |
| 静态资源 | ⚠️ 建议鉴权 | JS/CSS/图片等资源建议限制访问 |
| 数据库端口 | ❌ 禁止公网访问 | PostgreSQL 5432 端口不可暴露在公网 |
| Redis 端口 | ❌ 禁止公网访问 | Redis 6379 端口不可暴露在公网 |
| MinIO 端口 | ❌ 禁止公网访问 | MinIO 9000 端口不可暴露在公网 |

### 3.2 安全配置检查清单

- [ ] 确认防火墙仅开放必要端口（8080、5173）
- [ ] 确认数据库、Redis、MinIO 仅监听 localhost
- [ ] 确认所有 API 接口已配置 JWT 鉴权
- [ ] 确认 Swagger 文档已禁用或配置鉴权
- [ ] 确认前端路由已配置登录守卫
- [ ] 确认 CORS 配置已限制允许的来源
