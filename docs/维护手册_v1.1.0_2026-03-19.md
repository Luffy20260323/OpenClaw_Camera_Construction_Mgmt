## 4. 服务管理

### 4.1 服务启动

#### 4.1.1 启动 PostgreSQL

```bash
sudo systemctl start postgresql
sudo systemctl enable postgresql  # 开机自启
```

#### 4.1.2 启动 Redis

```bash
sudo systemctl start redis-server
sudo systemctl enable redis-server  # 开机自启
```

#### 4.1.3 启动 Nginx

```bash
sudo systemctl start nginx
sudo systemctl enable nginx  # 开机自启
```

#### 4.1.4 启动后端服务

```bash
cd /var/www/Camera_Construction_Project_Mgmt/backend

# 方式 1: 使用 JAR 包直接运行
nohup java -jar target/camera-lifecycle-system-1.0.0-SNAPSHOT.jar --server.port=8080 > /tmp/camera-backend.log 2>&1 &

# 方式 2: 使用 Maven 运行
mvn spring-boot:run

# 方式 3: 创建 systemd 服务（推荐）
sudo nano /etc/systemd/system/camera-backend.service
```

**systemd 服务配置** (`/etc/systemd/system/camera-backend.service`):

```ini
[Unit]
Description=Camera Lifecycle Management System Backend
After=syslog.target network.target postgresql.service redis.service

[Service]
Type=simple
User=www-data
WorkingDirectory=/var/www/Camera_Construction_Project_Mgmt/backend
ExecStart=/usr/bin/java -jar target/camera-lifecycle-system-1.0.0-SNAPSHOT.jar --server.port=8080
Restart=on-failure
RestartSec=10
StandardOutput=journal
StandardError=journal

[Install]
WantedBy=multi-user.target
```

```bash
# 重新加载 systemd 配置
sudo systemctl daemon-reload

# 启动服务
sudo systemctl start camera-backend
sudo systemctl enable camera-backend  # 开机自启
```

## 5. 日常维护

### 5.1 日志管理

#### 5.1.1 日志位置

| 服务 | 日志位置 |
|------|----------|
| **后端应用** | `/tmp/camera-backend.log` |
| **Nginx 访问日志** | `/var/log/nginx/access.log` |
| **Nginx 错误日志** | `/var/log/nginx/error.log` |
| **PostgreSQL 日志** | `/var/log/postgresql/postgresql-16-main.log` |
| **Redis 日志** | `/var/log/redis/redis-server.log` |
| **Systemd 日志** | `journalctl -u <service-name>` |

#### 5.1.2 日志查看

```bash
# 查看后端日志（实时）
tail -f /tmp/camera-backend.log

# 查看后端日志（最后 100 行）
tail -100 /tmp/camera-backend.log

# 查看 Nginx 错误日志
tail -f /var/log/nginx/error.log

# 查看 Systemd 服务日志
journalctl -u camera-backend -f

# 搜索特定错误
grep "ERROR" /tmp/camera-backend.log
grep "Exception" /tmp/camera-backend.log
```

#### 5.1.3 日志轮转

配置 `/etc/logrotate.d/camera-system`:

```
/tmp/camera-backend.log {
    daily
    rotate 30
    compress
    delaycompress
    missingok
    notifempty
    create 0640 www-data www-data
    postrotate
        systemctl reload camera-backend > /dev/null 2>&1 || true
    endscript
}
```

### 5.2 数据库维护

#### 5.2.1 数据库连接检查

```bash
# 切换到 postgres 用户
sudo -i -u postgres

# 进入 psql
psql

# 查看数据库列表
\l

# 连接到 camera_system 数据库
\c camera_system

# 查看表列表
\dt

# 退出
\q
```

#### 5.2.2 数据库备份

```bash
# 备份整个数据库
pg_dump -U postgres camera_system > /backup/camera_system_$(date +%Y%m%d_%H%M%S).sql

# 备份为压缩格式
pg_dump -U postgres camera_system | gzip > /backup/camera_system_$(date +%Y%m%d_%H%M%S).sql.gz

# 仅备份结构
pg_dump -U postgres -s camera_system > /backup/camera_system_schema_$(date +%Y%m%d_%H%M%S).sql

# 仅备份数据
pg_dump -U postgres -a camera_system > /backup/camera_system_data_$(date +%Y%m%d_%H%M%S).sql
```

#### 5.2.3 数据库恢复

```bash
# 恢复数据库
psql -U postgres camera_system < /backup/camera_system_20260314_120000.sql

# 恢复压缩备份
gunzip -c /backup/camera_system_20260314_120000.sql.gz | psql -U postgres camera_system
```

#### 5.2.4 数据库优化

```bash
# 清理空闲空间
psql -U postgres -c "VACUUM camera_system;"

# 分析表统计信息
psql -U postgres -c "ANALYZE camera_system;"

# 清理并分析
psql -U postgres -c "VACUUM ANALYZE camera_system;"
```

#### 5.2.5 数据库监控

```sql
-- 查看数据库大小
SELECT pg_size_pretty(pg_database_size('camera_system'));

-- 查看表大小
SELECT 
    tablename,
    pg_size_pretty(pg_total_relation_size(schemaname||'.'||tablename)) AS size
FROM pg_tables
WHERE schemaname = 'public'
ORDER BY pg_total_relation_size(schemaname||'.'||tablename) DESC;

-- 查看连接数
SELECT count(*) FROM pg_stat_activity;

-- 查看慢查询
SELECT * FROM pg_stat_statements ORDER BY total_time DESC LIMIT 10;
```

## 6. 故障排查

### 6.1 常见问题排查流程

```
1. 确认问题现象
   ↓
2. 检查服务状态（systemctl status）
   ↓
3. 查看日志文件（tail -f）
   ↓
4. 检查端口监听（netstat/ss）
   ↓
5. 检查进程状态（ps aux）
   ↓
6. 检查资源使用（top/htop）
   ↓
7. 检查网络连接（ping/curl）
   ↓
8. 检查数据库连接（psql/redis-cli）
```

### 6.2 无法访问系统

#### 6.2.1 现象

- 浏览器显示"无法访问此网站"
- 浏览器显示"502 Bad Gateway"
- 浏览器显示"503 Service Unavailable"

#### 6.2.2 排查步骤

```bash
# 1. 检查 Nginx 状态
systemctl status nginx

# 2. 检查后端服务状态
systemctl status camera-backend

# 3. 检查端口监听
ss -tlnp | grep -E "(80|8080)"

# 4. 测试后端 API
curl http://localhost:8080/api/health

# 5. 查看 Nginx 错误日志
tail -100 /var/log/nginx/error.log

# 6. 查看后端日志
tail -100 /tmp/camera-backend.log
```

#### 6.2.3 解决方案

**Nginx 未启动**:
```bash
sudo systemctl start nginx
```

**后端服务未启动**:
```bash
sudo systemctl start camera-backend
```

**端口冲突**:
```bash
# 查看占用端口的进程
sudo lsof -i :8080
sudo lsof -i :80

# 杀死占用进程
sudo kill -9 <PID>
```

**配置文件错误**:
```bash
# 测试 Nginx 配置
sudo nginx -t

# 重新加载配置
sudo nginx -s reload
```

- 后端日志显示"Authentication failed"
- 系统提示"数据库连接失败"

#### 6.5.2 排查步骤

```bash
# 1. 检查 PostgreSQL 服务
systemctl status postgresql

# 2. 检查 PostgreSQL 监听
ss -tlnp | grep 5432

# 3. 测试数据库连接
psql -U postgres -h localhost -p 5432

# 4. 查看 PostgreSQL 日志
tail -100 /var/log/postgresql/postgresql-16-main.log

# 5. 检查数据库配置
cat /var/www/Camera_Construction_Project_Mgmt/backend/src/main/resources/application.yml
```

#### 6.5.3 解决方案

**PostgreSQL 未启动**:
```bash
sudo systemctl start postgresql
```

**认证失败**:
```bash
# 修改 postgres 用户密码
sudo -i -u postgres
psql
ALTER USER postgres WITH PASSWORD 'new_password';
\q
```

**连接数超限**:
```sql
-- 查看最大连接数
SHOW max_connections;

-- 查看当前连接数
SELECT count(*) FROM pg_stat_activity;

-- 修改最大连接数（需要重启）
ALTER SYSTEM SET max_connections = 200;
```

|----------|----------|----------|----------|
| **数据库** | 每日 | 30 天 | pg_dump 全量备份 |
| **配置文件** | 每周 | 90 天 | 文件复制 |
| **日志文件** | 每月 | 180 天 | 压缩归档 |
| **前端代码** | 每次发布 | 永久 | Git 版本控制 |
| **后端代码** | 每次发布 | 永久 | Git 版本控制 |

### 7.2 备份脚本

创建 `/usr/local/bin/camera-backup.sh`:

```bash
#!/bin/bash
set -e

# 配置
BACKUP_DIR="/backup/camera_system"
DATE=$(date +%Y%m%d_%H%M%S)
DB_NAME="camera_system"
DB_USER="postgres"
RETENTION_DAYS=30

# 创建备份目录
mkdir -p "$BACKUP_DIR"

echo "=== 开始备份 ==="
echo "备份时间：$DATE"

# 备份数据库
echo "备份数据库..."
pg_dump -U $DB_USER $DB_NAME | gzip > "$BACKUP_DIR/db_$DATE.sql.gz"

# 备份配置文件
echo "备份配置文件..."
tar -czf "$BACKUP_DIR/config_$DATE.tar.gz" \
    /etc/nginx/sites-available/camera-system \
    /etc/postgresql/16/main/postgresql.conf \
    /etc/redis/redis.conf \
    /var/www/Camera_Construction_Project_Mgmt/backend/src/main/resources/application.yml \
    2>/dev/null || true

# 清理旧备份
echo "清理 $RETENTION_DAYS 天前的备份..."
find "$BACKUP_DIR" -name "*.gz" -mtime +$RETENTION_DAYS -delete

echo "=== 备份完成 ==="
echo "备份文件："
ls -lh "$BACKUP_DIR"/*_$DATE.*
```

```bash
# 添加执行权限
chmod +x /usr/local/bin/camera-backup.sh
```
