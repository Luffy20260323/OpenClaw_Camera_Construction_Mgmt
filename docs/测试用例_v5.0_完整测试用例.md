### 优先级定义

- **P0**: 核心功能，必须测试，阻塞性问题
- **P1**: 重要功能，应该测试
- **P2**: 次要功能，可选测试

### 测试环境要求

1. 数据库：MySQL 8.0+
2. 后端：Node.js 18+
3. 前端：现代浏览器（Chrome、Firefox、Edge）
4. 测试数据：基础数据（公司、角色、用户）已预置

### 测试数据准备

1. 创建测试公司（甲方、乙方、监理各 1 家）
2. 创建测试作业区（甲方公司下 2-3 个）
3. 创建测试用户（各角色至少 1 个）
4. 创建测试项目（施工类、技术服务类各 1 个）
5. 创建测试标段和点位

### 测试执行顺序建议

1. 先执行系统管理模块（TC-SY）测试，配置基础数据
2. 执行用户管理模块（TC-UM）测试，创建测试用户
3. 执行甲方项目管理模块（TC-PM）测试，创建项目和点位
4. 执行乙方施工管理模块（TC-CO）测试，模拟施工流程
5. 执行监理管理模块（TC-SU）测试，模拟监理审核
6. 执行项目可视化模块（TC-VIS）测试，验证沙盘功能
7. 执行零部件管理模块（TC-PC）测试，验证更换流程
