## 一、业务背景

### 1.1 业务场景举例

一个标段有 100 个视频监控点位需要建设，其中：
- **监控杆**: 30 个点位用 9 米杆，70 个用 12 米杆
- **摄像头**: 10 个点位配置 2 个摄像头，90 个配置 1 个摄像头
- **蓄电池**: 40 个点位配置 600AH，60 个配置 400AH
- **通信设备**: 20 个用 AI 边缘盒子、20 个用存储网关、60 个用路由器
- **声光报警器**: 50 个点位配置，50 个不配置

**设计目标**: 支持批量配置点位设备，减少重复数据录入，节省存储空间。

## 二、核心概念

### 2.1 概念层级图

```
零部件种类（ComponentType）
    │
    ├── 零部件属性集（ComponentAttrSet）
    │       │
    │       └── 属性集属性定义（ComponentAttrSetAttr）
    │               │
    │               └── 零部件属性集实例（ComponentAttrSetInstance）
    │                       │
    │                       └── 零部件实例（ComponentInstance）
    │
    └── 点位设备模型（PointDeviceModel）
            │
            ├── 设备模型零部件项（PointDeviceModelItem）
            │
            └── 点位设备模型实例（PointDeviceModelInstance）
                    │
                    └── 点位引用（Point）
```

### 2.2 概念说明

| 概念 | 说明 | 创建者 | 示例 |
|------|------|--------|------|
| **零部件种类** | 设备类型分类 | 系统管理员 | 监控杆、摄像头、光伏板、蓄电池... |
| **零部件属性集** | 每种零部件的属性模板 | 系统管理员 | "9米监控杆属性集"、"12米监控杆属性集" |
| **属性集属性定义** | 属性集中包含的具体属性 | 系统管理员 | 厂商、品牌、型号、高度、材质... |
| **零部件属性集实例** | 属性集的具体值 | 系统管理员 | "海康威视9米监控杆实例" |
| **零部件实例** | 实际的零部件对象 | 系统自动生成 | 监控杆实例 #001 |
| **点位设备模型** | 点位设备配置模板 | 系统管理员 | "标准监控点位模型" |
| **设备模型零部件项** | 模型中包含的零部件种类和数量 | 系统管理员 | 监控杆×1、摄像头×1、蓄电池×1 |
| **点位设备模型实例** | 包含具体零部件实例 ID 的配置 | 系统管理员 | "30个点位用9米杆配置实例" |

## 三、数据模型设计

### 3.1 数据库表结构

#### 3.1.1 零部件种类表 (component_types)

```sql
CREATE TABLE component_types (
    id BIGSERIAL PRIMARY KEY,
    name VARCHAR(100) NOT NULL,           -- 种类名称
    code VARCHAR(50) NOT NULL UNIQUE,     -- 种类编码
    description TEXT,                      -- 描述
    sequence_no INT DEFAULT 10,           -- 显示序号
    is_active BOOLEAN DEFAULT true,       -- 是否启用
    created_by BIGINT,                    -- 创建人
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

COMMENT ON TABLE component_types IS '零部件种类表';
COMMENT ON COLUMN component_types.name IS '种类名称：监控杆、摄像头、光伏板等';
COMMENT ON COLUMN component_types.code IS '种类编码：MONITORING_POLE、CAMERA等';
COMMENT ON COLUMN component_types.sequence_no IS '显示序号，用于前端排序';
```

#### 3.1.2 零部件属性集表 (component_attr_sets)

```sql
CREATE TABLE component_attr_sets (
    id BIGSERIAL PRIMARY KEY,
    component_type_id BIGINT NOT NULL,    -- 关联零部件种类
    name VARCHAR(100) NOT NULL,           -- 属性集名称
    code VARCHAR(50) NOT NULL,            -- 属性集编码
    description TEXT,                      -- 描述
    sequence_no INT DEFAULT 10,           -- 显示序号
    is_active BOOLEAN DEFAULT true,       -- 是否启用
    created_by BIGINT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    CONSTRAINT fk_attr_set_type FOREIGN KEY (component_type_id) 
        REFERENCES component_types(id) ON DELETE CASCADE
);

COMMENT ON TABLE component_attr_sets IS '零部件属性集表';
COMMENT ON COLUMN component_attr_sets.name IS '属性集名称：如"9米监控杆属性集"';
```

#### 3.1.3 属性集属性定义表 (component_attr_set_attrs)

```sql
CREATE TABLE component_attr_set_attrs (
    id BIGSERIAL PRIMARY KEY,
    attr_set_id BIGINT NOT NULL,          -- 关联属性集
    attr_name VARCHAR(100) NOT NULL,      -- 属性名称
    attr_code VARCHAR(50) NOT NULL,       -- 属性编码（数据库字段名）
    attr_type VARCHAR(20) NOT NULL,       -- 属性类型：TEXT/NUMBER/DATE/SELECT...
    unit VARCHAR(50),                      -- 单位
    options JSONB,                         -- 选项值（用于 SELECT 类型）
    default_value TEXT,                    -- 默认值
    sequence_no INT DEFAULT 10,           -- 显示序号
    is_required BOOLEAN DEFAULT false,    -- 是否必填
    validation_rule TEXT,                  -- 验证规则
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    CONSTRAINT fk_attr_set FOREIGN KEY (attr_set_id) 
        REFERENCES component_attr_sets(id) ON DELETE CASCADE
);

COMMENT ON TABLE component_attr_set_attrs IS '属性集属性定义表';
COMMENT ON COLUMN component_attr_set_attrs.attr_type IS 'TEXT/NUMBER/DATE/SELECT/MULTI_SELECT/FILE';
```

#### 3.1.4 零部件属性集实例表 (component_attr_set_instances)

```sql
CREATE TABLE component_attr_set_instances (
    id BIGSERIAL PRIMARY KEY,
    attr_set_id BIGINT NOT NULL,          -- 关联属性集
    name VARCHAR(200) NOT NULL,           -- 实例名称
    attr_values JSONB NOT NULL,           -- 属性值 JSON
    created_by BIGINT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    CONSTRAINT fk_attr_set_instance FOREIGN KEY (attr_set_id) 
        REFERENCES component_attr_sets(id) ON DELETE CASCADE
);

COMMENT ON TABLE component_attr_set_instances IS '零部件属性集实例表';
COMMENT ON COLUMN component_attr_set_instances.attr_values IS '属性值JSON：{"厂商":"海康威视","高度":"9米","材质":"热镀锌钢"}';
```

**attr_values JSON 结构示例**:
```json
{
  "manufacturer": "海康威视",
  "brand": "海康",
  "model": "HK-POLE-9M",
  "height": 9,
  "material": "热镀锌钢",
  "diameter": 140,
  "weight": 150.5
}
```

#### 3.1.5 零部件实例表 (component_instances)

```sql
CREATE TABLE component_instances (
    id BIGSERIAL PRIMARY KEY,
    component_type_id BIGINT NOT NULL,    -- 零部件种类
    attr_set_instance_id BIGINT,          -- 属性集实例（可选）
    serial_number VARCHAR(100),           -- 序列号
    status VARCHAR(20) DEFAULT 'normal',  -- 状态：normal/replaced/scrapped
    created_by BIGINT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    CONSTRAINT fk_instance_type FOREIGN KEY (component_type_id) 
        REFERENCES component_types(id),
    CONSTRAINT fk_instance_attr FOREIGN KEY (attr_set_instance_id) 
        REFERENCES component_attr_set_instances(id)
);

COMMENT ON TABLE component_instances IS '零部件实例表';
COMMENT ON COLUMN component_instances.status IS 'normal:正常使用, replaced:已更换, scrapped:已报废';
```

#### 3.1.6 点位设备模型表 (point_device_models)

```sql
CREATE TABLE point_device_models (
    id BIGSERIAL PRIMARY KEY,
    name VARCHAR(200) NOT NULL,           -- 模型名称
    code VARCHAR(50),                     -- 模型编码
    description TEXT,                      -- 描述
    project_id BIGINT,                    -- 关联项目（可选）
    created_by BIGINT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

COMMENT ON TABLE point_device_models IS '点位设备模型表';
COMMENT ON COLUMN point_device_models.name IS '模型名称：如"标准监控点位模型"';
```

#### 3.1.7 设备模型零部件项表 (point_device_model_items)

```sql
CREATE TABLE point_device_model_items (
    id BIGSERIAL PRIMARY KEY,
    model_id BIGINT NOT NULL,             -- 关联设备模型
    component_type_id BIGINT NOT NULL,    -- 零部件种类
    quantity INT DEFAULT 1,               -- 数量
    sequence_no INT DEFAULT 10,           -- 显示序号
    is_required BOOLEAN DEFAULT true,     -- 是否必选
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    CONSTRAINT fk_model_item_model FOREIGN KEY (model_id) 
        REFERENCES point_device_models(id) ON DELETE CASCADE,
    CONSTRAINT fk_model_item_type FOREIGN KEY (component_type_id) 
        REFERENCES component_types(id)
);

COMMENT ON TABLE point_device_model_items IS '设备模型零部件项表';
COMMENT ON COLUMN point_device_model_items.quantity IS '该种类零部件的数量';
```

#### 3.1.8 点位设备模型实例表 (point_device_model_instances)

```sql
CREATE TABLE point_device_model_instances (
    id BIGSERIAL PRIMARY KEY,
    model_id BIGINT NOT NULL,             -- 关联设备模型
    name VARCHAR(200) NOT NULL,           -- 实例名称
    component_instances JSONB,            -- 零部件实例映射
    created_by BIGINT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    CONSTRAINT fk_model_instance FOREIGN KEY (model_id) 
        REFERENCES point_device_models(id)
);

COMMENT ON TABLE point_device_model_instances IS '点位设备模型实例表';
COMMENT ON COLUMN point_device_model_instances.component_instances IS '零部件实例映射JSON';
```

**component_instances JSON 结构示例**:
```json
{
  "component_instances": [
    {
      "component_type_id": 1,
      "component_type_name": "监控杆",
      "instances": [
        {"instance_id": 101, "sequence_no": 10}
      ]
    },
    {
      "component_type_id": 2,
      "component_type_name": "摄像头",
      "instances": [
        {"instance_id": 201, "sequence_no": 10},
        {"instance_id": 202, "sequence_no": 20}
      ]
    }
  ]
}
```

#### 3.1.9 点位表关联 (points 表新增字段)

```sql
-- 在现有 points 表中新增字段
ALTER TABLE points ADD COLUMN device_model_instance_id BIGINT;
ALTER TABLE points ADD CONSTRAINT fk_point_model_instance 
    FOREIGN KEY (device_model_instance_id) REFERENCES point_device_model_instances(id);

COMMENT ON COLUMN points.device_model_instance_id IS '点位设备模型实例ID';
```

## 四、业务流程设计

### 4.1 零部件种类管理流程

```
系统管理员 → 创建零部件种类 → 定义属性集 → 定义属性
    ↓
启用种类 → 可用于设备模型配置
```

### 4.2 属性集实例创建流程

```
系统管理员 → 选择属性集 → 填写属性值 → 保存实例
    ↓
实例可被多个零部件实例引用
```

### 4.3 点位设备配置流程

```
系统管理员 → 创建设备模型 → 添加零部件种类和数量
    ↓
创建设备模型实例 → 为每个零部件种类选择属性集实例
    ↓
生成零部件实例 → 实例被多个点位引用
```

### 4.4 点位引用设备模型实例流程

```
甲方管理员 → 选择点位范围 → 选择设备模型实例 → 批量关联
    ↓
点位通过 device_model_instance_id 引用实例
```

### 9.2 并行性分析

```
【第一阶段】P0 任务（可并行）
├── subagent-1: 数据库迁移脚本 + 零部件种类后端
├── subagent-2: 零部件属性集后端
```

## 十一、附录

### 11.1 与场景故事方案对比

| 维度 | 本方案 | 场景故事方案 |
|------|--------|-------------|
| 属性存储 | JSONB 字段（静态表） | 动态表创建 |
| 维护难度 | ✅ 简单 | ⚠️ 复杂（需管理动态表元数据） |
| 存储效率 | ✅ 属性集实例复用 | ✅ 模板复用+差异存储 |
| 批量配置 | ✅ 设备模型实例可被多点位引用 | ⚠️ 每个点位独立配置 |
| 概念清晰度 | ✅ 层级分明，易理解 | ⚠️ 动态表概念较抽象 |

### 11.2 设计优势

1. **存储效率高**: 属性集实例复用，节省存储空间
2. **维护简单**: 传统静态表，无需管理动态表
3. **批量配置**: 设备模型实例可被多点位引用
4. **概念清晰**: 层级分明，易于理解和维护
5. **扩展性好**: JSONB 字段支持灵活扩展新属性
