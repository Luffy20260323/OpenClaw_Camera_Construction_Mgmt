1. 安装 OpenVPN

1更新软件包列表并安装 OpenVPN

sudo apt update && sudo apt install -y openvpn
✅ 安装完成后，OpenVPN 服务会自动注册到 systemd

2. 准备配置文件

2创建 PKI 目录存放证书文件

mkdir -p /etc/openvpn/pki

3从服务器配置目录复制证书文件

# 服务器 1 证书
cp /root/openvpn_conf/ov_srv1/ca.crt /etc/openvpn/pki/ov1_ca.crt
cp /root/openvpn_conf/ov_srv1/cloudsrv_00002.crt /etc/openvpn/pki/ov1_client.crt
cp /root/openvpn_conf/ov_srv1/cloudsrv_00002.key /etc/openvpn/pki/ov1_client.key
cp /root/openvpn_conf/ov_srv1/ta.key /etc/openvpn/pki/ov1_ta.key

# 服务器 2 证书
cp /root/openvpn_conf/ov_srv2/ca.crt /etc/openvpn/pki/ov2_ca.crt
cp /root/openvpn_conf/ov_srv2/cloudsrv_00002.crt /etc/openvpn/pki/ov2_client.crt
cp /root/openvpn_conf/ov_srv2/cloudsrv_00002.key /etc/openvpn/pki/ov2_client.key
cp /root/openvpn_conf/ov_srv2/ta.key /etc/openvpn/pki/ov2_ta.key

3. 配置客户端 1

4创建客户端 1 配置文件

cat > /etc/openvpn/client/client_ov1.conf << 'EOF'
# OpenVPN Client 1 - Server ov.bjqd.net.cn
client
dev tun
proto udp
remote ov.bjqd.net.cn 1194
resolv-retry infinite
nobind
persist-key
persist-tun

# SSL/TLS certificates (relative paths)
ca ../pki/ov1_ca.crt
cert ../pki/ov1_client.crt
key ../pki/ov1_client.key
tls-auth ../pki/ov1_ta.key 1

# Security
remote-cert-tls server

# Logging
verb 3
EOF

关键参数说明

参数 说明
client 指定为客户端模式
dev tun 使用 TUN 设备（三层路由）
proto udp 使用 UDP 协议（也可用 tcp）
remote 服务器地址和端口
tls-auth TLS 认证密钥，1 表示客户端方向

4. 配置客户端 2

5创建客户端 2 配置文件（与客户端 1 类似，修改服务器地址和证书）

cat > /etc/openvpn/client/client_ov2.conf << 'EOF'
# OpenVPN Client 2 - Server ov2.bjqd.net.cn
client
dev tun
proto udp
remote ov2.bjqd.net.cn 1194
resolv-retry infinite
nobind
persist-key
persist-tun

# SSL/TLS certificates (relative paths)
ca ../pki/ov2_ca.crt
cert ../pki/ov2_client.crt
key ../pki/ov2_client.key
tls-auth ../pki/ov2_ta.key 1

# Security
remote-cert-tls server

# Logging
verb 3
EOF

5. 启动服务

6启用开机自启

systemctl enable openvpn-client@client_ov1.service
systemctl enable openvpn-client@client_ov2.service

7启动服务

systemctl start openvpn-client@client_ov1.service
systemctl start openvpn-client@client_ov2.service

8检查服务状态

systemctl status openvpn-client@client_ov1.service openvpn-client@client_ov2.service
✅ 成功连接后，状态应显示 Active: active (running) 和 Initialization Sequence Completed

6. 文件结构

/etc/openvpn/
  • client/
    • client_ov1.conf — 客户端 1 配置
    • client_ov2.conf — 客户端 2 配置
  • pki/
    • ov1_ca.crt — 服务器 1 CA 证书
    • ov1_client.crt — 服务器 1 客户端证书
    • ov1_client.key — 服务器 1 客户端密钥
    • ov1_ta.key — 服务器 1 TLS 认证密钥
    • ov2_ca.crt — 服务器 2 CA 证书
    • ov2_client.crt — 服务器 2 客户端证书
    • ov2_client.key — 服务器 2 客户端密钥
    • ov2_ta.key — 服务器 2 TLS 认证密钥

7. 故障排查

问题 1：域名无法解析

症状：日志显示 RESOLVE: Cannot resolve host address

解决方案：

# 检查 DNS 解析
nslookup your-vpn-server.com

# 如果域名不存在，使用 IP 地址或修正域名
# 编辑配置文件，修改 remote 行
sed -i 's/old.domain.com/new.domain.com/' /etc/openvpn/client/client_xxx.conf

# 重启服务
systemctl restart openvpn-client@client_xxx.service

问题 2：连接失败

症状：服务反复重启，无法完成初始化

检查步骤：

# 查看详细日志
journalctl -u openvpn-client@client_xxx.service -f

# 检查证书文件权限
ls -la /etc/openvpn/pki/

# 测试手动连接
sudo openvpn --config /etc/openvpn/client/client_xxx.conf

问题 3：路由冲突

症状：两个客户端都连接成功，但网络访问异常

解决方案：检查路由表，确保两个 VPN 的路由不冲突

ip route show
ip addr show
