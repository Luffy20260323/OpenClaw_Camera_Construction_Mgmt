## 🔒 权限说明

### 当前配置

只有以下用户可以访问文档中心：

| 用户/角色 | 权限 | 说明 |
|----------|------|------|
| admin 用户 | ✅ 有权限 | 系统管理员 |
| ROLE_SYSTEM_ADMIN | ✅ 有权限 | 系统管理员角色 |

### 如何授权其他用户

1. 登录系统（使用 admin 账号）
2. 进入 **系统管理 → 角色权限配置**
3. 选择要授权的角色
4. 在权限列表中找到并勾选 **`system:docs:view`**（查看系统文档）
5. 保存配置

## 📁 添加新文档

### 方式 1：直接上传到服务器

```bash
# 1. 将文档文件复制到 docs 目录
cp your-document.pdf /root/.openclaw/workspace/OpenClaw_Camera_Construction_Mgmt/docs/

# 2. 文档会自动出现在文档中心列表中
# 系统会自动扫描以下格式：
# - .html (HTML 文档)
# - .md (Markdown 文档)
# - .pdf (PDF 文档)
```

### 方式 2：通过 Git 提交

```bash
# 1. 将文档添加到项目 docs 目录
cd /root/.openclaw/workspace/OpenClaw_Camera_Construction_Mgmt
cp your-document.pdf docs/

# 2. 提交到 Git
git add docs/your-document.pdf
git commit -m "docs: 添加新文档 - your-document"
git push

# 3. 重新构建前端（如需要）
docker compose build frontend
docker compose restart frontend
```

### 文档元数据

系统会自动从文档内容提取：
- **标题**：从 Markdown 的 `# 标题` 或 HTML 的 `<h1>` 标签
- **描述**：从副标题或第一段文字
- **分类**：根据文件名自动判断
- **文件大小**：自动计算
- **更新时间**：文件最后修改时间

## ❓ 常见问题

### Q: 为什么我看不到文档中心菜单？

A: 检查以下几点：
1. 是否已登录系统
2. 当前用户是否有 `system:docs:view` 权限
3. 尝试退出重新登录

### Q: 文档内容不显示怎么办？

A: 
1. 检查文件是否在正确的目录
2. 确认文件格式支持（html/md/pdf）
3. 查看浏览器控制台是否有错误
4. 尝试强制刷新页面（Ctrl+Shift+R）

### Q: 如何修改文档？

A:
1. 直接修改服务器上的文件
2. 或通过 Git 更新后重新部署
3. 修改后无需重启服务，实时生效
