## 🔐 核心功能实现

### 1. 数据范围类型

```sql
-- 4 种标准数据范围
ALL      - 全部数据（不过滤）
COMPANY  - 本公司数据
WORKAREA - 本作业区数据
SELF     - 仅个人数据

-- 支持自定义范围
CUSTOM   - 自定义数据范围（JSON 配置）
```

### 2. 数据权限模板

```sql

-- COMPANY_DATA  | 本公司数据    | COMPANY
-- WORKAREA_DATA | 本作业区数据  | WORKAREA
-- SELF_DATA     | 仅个人数据    | SELF
```

### 3. 用户数据权限

```sql
-- 查询用户数据权限
SELECT 
    u.username,
    udp.data_scope_type,
    udp.company_id,
    udp.workarea_ids,
    udp.source,
    udp.is_fixed
FROM users u
JOIN user_data_permissions udp ON u.id = udp.user_id;
```

### 4. 数据权限函数

```sql
-- 获取用户数据权限 SQL 条件
SELECT get_user_data_permission_sql(1, 'user');

-- 结果示例：
-- ALL:      "1=1"
-- COMPANY:  "company_id = 1"
-- WORKAREA: "workarea_id IN (1,2,3)"
-- SELF:     "created_by = 1"
```

### 5. 后端集成示例

```java
// MyBatis 拦截器自动注入数据权限
@Intercepts({
    @Signature(type = Executor.class, method = "query", args = {...})
})
public class DataPermissionInterceptor implements Interceptor {
    
    @Override
    public Object intercept(Invocation invocation) throws Throwable {
        // 获取当前用户
        Long userId = UserContext.getCurrentUserId();
        
        // 获取用户数据权限
        String permissionSql = get_user_data_permission_sql(userId, entityType);
        
        // 自动拼接到 SQL
        String originalSql = (String) invocation.getArgs()[0];
        String newSql = originalSql + " AND " + permissionSql;
        
        invocation.getArgs()[0] = newSql;
        return invocation.proceed();
    }
}
```
