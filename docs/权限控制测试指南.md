## 常见问题

### Q1：修改权限后不生效？

**A：** 必须重新登录！权限信息存储在 localStorage 中，只有重新登录才会刷新。

### Q2：页面显示 403？

**A：** 说明当前用户没有该菜单的访问权限。联系系统管理员配置权限。

### Q3：API 返回 403？

**A：** 后端权限过滤器拦截了越权访问。检查用户是否有对应菜单权限。

### Q4：如何查看用户当前权限？

**A：** 
1. 系统管理员进入"用户权限配置"页面
2. 选择用户查看权限列表
3. 权限来源列显示"角色默认"或"自定义"

## 权限配置建议

### 最小权限原则
- 默认只授予必要权限
- 需要额外权限时单独配置
- 定期审查用户权限

### 角色规划
- 系统管理员：仅限运维人员
- 公司管理员：各公司负责人
- 项目经理：项目管理需要
- 普通用户：仅查看个人信息

### 审计建议
- 定期检查用户权限配置
- 记录权限变更日志
- 离职/转岗及时回收权限

## 技术支持

如遇问题，请检查：
1. 浏览器缓存是否清除
2. 是否重新登录
3. 后端日志：`docker logs camera-backend`
4. 前端控制台：F12 → Console
5. 数据库权限配置：
```sql
SELECT * FROM user_menu_permissions WHERE user_id = <用户 ID>;
SELECT * FROM role_menu_permissions WHERE role_id = <角色 ID>;
```
