# 菜单权限管理系统

## 概述

实现了完整的权限控制系统，支持系统管理员为每个用户定制菜单访问权限。权限控制覆盖路由、页面、按钮和 API 四个层级。

## 核心特性

### 1. 权限层级

- **角色默认权限**：根据用户角色自动分配的基础菜单权限
- **自定义权限**：系统管理员手动配置的用户特定权限（优先级高于角色默认）

### 2. 权限优先级

```
用户最终权限 = 自定义权限（如果有）OR 角色默认权限
```

### 3. 权限控制层级

| 层级 | 控制点 | 实现方式 |
|------|--------|----------|
| **路由级** | 页面访问 | 路由守卫验证菜单权限，无权限重定向 |
| **页面级** | 页面内容 | 页面组件检查权限，无权限提示 |
| **按钮级** | 操作按钮 | 指令控制按钮显示/隐藏 |
| **API 级** | 后端接口 | 过滤器防止越权访问 |

## 数据库表结构

### menus - 菜单定义表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| menu_code | VARCHAR(50) | 菜单编码（唯一） |
| menu_name | VARCHAR(100) | 菜单名称 |
| menu_path | VARCHAR(255) | 菜单路径 |
| parent_id | BIGINT | 父菜单 ID |
| sort_order | INT | 排序 |
| icon | VARCHAR(50) | 图标 |
| is_visible | BOOLEAN | 是否可见 |
| required_permission | VARCHAR(100) | 所需权限 |
| is_system_protected | BOOLEAN | 是否系统保护 |

### role_menu_permissions - 角色菜单权限表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| role_id | BIGINT | 角色 ID |
| menu_id | BIGINT | 菜单 ID |
| can_view | BOOLEAN | 是否可查看 |
| can_operate | BOOLEAN | 是否可操作 |

### user_menu_permissions - 用户自定义菜单权限表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| user_id | BIGINT | 用户 ID |
| menu_id | BIGINT | 菜单 ID |
| can_view | BOOLEAN | 是否可查看 |
| can_operate | BOOLEAN | 是否可操作 |
| granted_by | BIGINT | 授权人 ID |
| granted_at | TIMESTAMP | 授权时间 |

## 前端使用

### 权限指令

在模板中使用权限指令控制元素显示：

```vue
<template>
  <!-- 检查菜单权限 -->
  <el-button v-menu-permission="'user_management'">用户管理</el-button>
  
  <!-- 检查操作权限 -->
  <el-button v-menu-operate="'user_management'">操作用户管理</el-button>
  
  <!-- 检查权限点 -->
  <el-button v-permission="'user:create'">创建用户</el-button>
</template>
```

### 权限函数

在脚本中使用权限函数：

```javascript
import { hasMenuPermission, hasPermission } from '@/utils/permission'

// 检查菜单权限
if (hasMenuPermission('user_management')) {
  // 有权限
}

// 检查权限点
if (hasPermission('user:create')) {
  // 有权限
}
```

## API 接口

### 获取用户菜单列表

```http
GET /api/menu/my-menus
Authorization: Bearer {token}
```

返回当前用户有权访问的菜单列表。

### 获取所有菜单（系统管理员）

```http
GET /api/menu/all
Authorization: Bearer {token}
```

### 获取用户菜单权限详情（系统管理员）

```http
GET /api/menu/user-permissions/{userId}
Authorization: Bearer {token}
```

返回用户的菜单权限配置，包含权限来源（ROLE 或 CUSTOM）。

### 更新用户菜单权限（系统管理员）

```http
PUT /api/menu/user-permission
Authorization: Bearer {token}
Content-Type: application/json

{
  "userId": 1,
  "menuId": 2,
  "canView": true,
  "canOperate": true
}
```

### 删除用户自定义权限（系统管理员）

```http
DELETE /api/menu/user-permission?userId=1&menuId=2
Authorization: Bearer {token}
```

删除自定义权限，恢复为角色默认权限。

## 前端页面

### 用户权限配置页面

路径：`/system/user-permission`

功能：
- 选择用户
- 查看用户当前菜单权限（区分角色默认和自定义）
- 修改菜单查看/操作权限
- 恢复为角色默认权限
- 批量保存修改

## 使用说明

### 系统管理员配置用户权限

1. 登录系统（系统管理员账号）
2. 进入"系统管理" → "用户权限配置"
3. 选择要配置的用户
4. 修改菜单权限开关
5. 点击"保存修改"

### 权限生效

- 用户重新登录后权限生效
- 权限变更实时记录到数据库
- 自定义权限优先级高于角色默认权限

## 测试场景

### 场景 1：普通用户越权访问

1. 使用普通用户（如 hz_admin21 的普通用户角色）登录
2. 浏览器地址栏输入 `/user/management`
3. **预期**：自动重定向到首页，显示无权限提示

### 场景 2：公司管理员权限

1. 使用甲方管理员（kunming001）登录
2. 查看菜单：应显示"用户管理"和"个人中心"
3. 尝试访问 `/role`（角色管理）
4. **预期**：自动重定向到首页

### 场景 3：系统管理员定制权限

1. 使用系统管理员（admin）登录
2. 进入"系统管理" → "用户权限配置"
3. 选择 kunming001 用户
4. 勾选"角色管理"菜单的查看权限
5. 保存修改
6. kunming001 重新登录
7. **预期**：可以看到并访问角色管理页面

### 场景 4：API 越权访问

1. 使用普通用户登录获取 token
2. 使用 curl 访问 `/api/user/list`
3. **预期**：返回 403 Forbidden

## 注意事项

1. **系统保护菜单**：个人中心等基础菜单权限不可修改
2. **权限冲突**：自定义权限覆盖角色默认权限
3. **删除自定义**：删除自定义权限后自动恢复角色默认
4. **审计日志**：所有权限修改记录授权人和时间
5. **缓存问题**：权限变更后需重新登录生效
6. **清除浏览器缓存**：前端更新后必须清除缓存
