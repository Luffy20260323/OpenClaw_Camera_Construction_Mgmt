## 一、项目概述

### 1.1 项目定位

摄像头生命周期管理系统是一个聚焦**项目交付过程管理**和**最终结算管理**的多角色协作平台。系统支持甲方、乙方、监理三方协作，实现项目从开工到施工、验收、结算、运维的全生命周期管理。

### 1.2 技术栈

| 层级 | 技术 | 版本 | 说明 |
|------|------|------|------|
| **后端框架** | Spring Boot | 3.2.4 | 核心后端框架 |
| **开发语言** | Java | 17 | JDK 17+ |
| **ORM 框架** | MyBatis-Plus | 3.5.5 | 数据持久层 |
| **数据库** | PostgreSQL | 16.x | 主数据库 |
| **缓存** | Redis | 7.x | 缓存/会话存储 |
| **文件存储** | MinIO | 8.5.9 | 对象存储 |
| **认证** | JWT + Spring Security | 0.12.5 | 令牌认证 |
| **API 文档** | Knife4j | 4.4.0 | Swagger 增强 |
| **前端框架** | Vue 3 | 3.x | 渐进式框架 |
| **构建工具** | Vite | 5.x | 前端构建 |
| **UI 组件** | Element Plus | 2.x | Vue3 组件库 |
| **状态管理** | Pinia | 2.x | Vue 状态管理 |

## 六、技术特点与发现

### 6.1 架构优点

1. **模块划分清晰** - 按业务领域划分模块（auth/user/company/role/workarea），便于维护和扩展
2. **统一布局组件** - AdminLayout 统一管理所有管理页面的导航和底部声明
3. **数据库迁移规范** - 使用 Flyway 进行版本化管理（V1/V2...）
4. **文档齐全** - 测试用例、用户手册、维护手册、场景故事等文档完整
5. **权限设计合理** - 基于公司类型的分级权限控制

### 6.2 技术债务/注意事项

1. **数据库迁移脚本较少** - 目前只有 V1 和 V2，后续模块需要新增 V3+ 脚本
2. **前端路由较简单** - 目前只有基础管理页面，后续需要扩展大量业务页面
3. **缺少项目管理相关表** - 需要新增项目、标段、点位、任务等核心业务表
4. **文件上传功能待完善** - MinIO 配置存在，但前端上传组件待开发

### 6.3 最新开发动态（2026-03-19）

- ✅ 用户批量导入 Excel 模板下载 + 文件上传功能重构
- ✅ 用户 - 作业区关联功能完善

## 七、开发建议

### 7.1 下一步开发优先级

根据 VERSION.md 和今日开发日志，建议按以下顺序开发：

1. **第一阶段：基础数据配置**
   - 项目大类/小类管理
   - 施工类型配置
   - 子任务模板配置

2. **第二阶段：项目核心模块**
   - 项目创建与标段划分
   - 点位分解与设备模型配置
   - 项目沙盘（地图可视化）

3. **第三阶段：施工管理流程**
   - 施工组织组建
   - 任务分配与数据填写
   - 多级审核流程

4. **第四阶段：监理与结算**
   - 监理审核与周报
   - 工程量统计
   - 进度款与最终结算

### 7.2 开发规范建议

- **后端：** 遵循现有模块结构，新增 module/project、module/task 等
