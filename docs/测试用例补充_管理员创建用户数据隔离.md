# 测试用例补充 - 管理员创建用户数据隔离测试

**版本号**: v5.0-Isolation  
**日期**: 2026-03-21  
**编制**: OpenClaw AI Assistant  
**补充说明**: 测试甲方/乙方/监理管理员创建用户时的公司数据隔离和角色权限限制  

---

## 🎯 测试目标

验证**各公司管理员创建用户时的数据隔离原则**：

1. **甲方管理员**创建用户 → 公司只能选择甲方公司，角色只能是甲方角色
2. **乙方管理员**创建用户 → 公司只能选择乙方公司，角色只能是乙方角色
3. **监理管理员**创建用户 → 公司只能选择监理公司，角色只能是监理角色
4. **系统管理员**创建用户 → 可以选择任何公司，角色根据选择的公司联动

## 📋 测试用例

### TC-ISO-001 甲方管理员创建用户 - 公司列表验证

| 项目 | 内容 |
|------|------|
| **模块** | 用户管理 - 数据隔离 |
| **功能点** | 甲方管理员创建用户时，公司列表仅显示甲方公司 |
| **测试类型** | 功能测试 + 安全测试 |
| **前置条件** | 1. 甲方管理员已登录（属于 test_company_jiafang_001）<br>2. 系统存在甲方、乙方、监理公司<br>3. 系统存在甲方、乙方、监理角色 |
| **测试步骤** | 1. 甲方管理员进入"公司用户管理"模块<br>2. 点击"新增用户"<br>3. 查看"所属公司"字段的状态和可选列表 |
| **预期结果** | 1. "所属公司"字段**自动填充**为当前管理员所属公司（test_company_jiafang_001）<br>2. 公司字段**不可编辑**或**下拉列表仅显示本公司**<br>3. **不显示**乙方公司<br>4. **不显示**监理公司<br>5. 无法通过前端修改用户所属公司 |
| **优先级** | P0 |
| **安全等级** | 🔒 高（数据隔离） |

## 🔐 安全测试要点

### 前端验证
- 公司字段自动填充且不可修改（公司管理员）
- 角色列表根据公司类型过滤
- 作业区列表根据本公司过滤

### 后端验证
- API 必须验证管理员的公司权限
- 禁止跨公司创建用户
- 返回适当的错误码（403 或 400）

### 数据库验证
- 创建的用户公司 ID 必须与管理员公司 ID 一致
- 角色必须属于正确的公司类型
- 作业区必须属于本公司
