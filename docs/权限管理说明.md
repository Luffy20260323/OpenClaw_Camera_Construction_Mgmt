## 🔐 权限控制逻辑

### 1. 公司管理权限

**只有系统管理员可以管理公司**

- 创建公司
- 更新公司信息
- 删除公司

**实现代码**:
```java
userService.validateCompanyManagementPermission(operatorId);
```

**错误提示**: "只有系统管理员才有权限管理公司"

### 2. 角色管理权限

- 删除角色

```java
userService.validateRoleManagementPermission(operatorId);
```

**错误提示**: "只有系统管理员才有权限管理角色"


**错误提示**: "只有系统管理员才有权限管理作业区"

### 4. 用户管理权限

#### 4.1 创建用户

**权限规则**:
- 系统管理员：可以创建所有公司类型的用户
- 甲方管理员：只能创建甲方公司的用户
- 乙方管理员：只能创建乙方公司的用户
- 监理方管理员：只能创建监理方公司的用户

**实现代码**:
```java
userService.validateUserCreationPermission(operatorId, targetCompanyTypeId);
```

**错误提示**: "您没有权限创建该公司类型的用户。系统管理员可创建所有用户，其他管理员只能创建本公司类型的用户"

#### 4.2 审批用户

**权限规则**:
- 系统管理员：可以审批所有公司类型的用户
- 甲方管理员：只能审批甲方公司的用户
- 乙方管理员：只能审批乙方公司的用户
- 监理方管理员：只能审批监理方公司的用户

**实现代码**:
```java
userService.validateApprovalPermission(operatorId, targetUserId);
```

**错误提示**: "您没有权限审批该公司类型的用户。系统管理员可审批所有用户，其他管理员只能审批本公司类型的用户"

## 📝 实现细节

### 公司类型常量定义

```java
private static final Integer COMPANY_TYPE_JIAFANG = 1;    // 甲方
private static final Integer COMPANY_TYPE_YIFANG = 2;     // 乙方
private static final Integer COMPANY_TYPE_JIANLIFANG = 3; // 监理方
private static final Integer COMPANY_TYPE_SYSTEM = 4;     // 系统管理员
```

### 权限校验流程

1. **获取操作人信息**
   ```java
   User operator = userMapper.selectById(operatorId);
   ```

2. **获取操作人的公司类型**
   ```java
   Company operatorCompany = companyMapper.selectById(operator.getCompanyId());
   Integer operatorCompanyTypeId = operatorCompany.getTypeId().intValue();
   ```

3. **判断权限**
   ```java
   // 系统管理员（公司类型 ID = 4）拥有所有权限
   if (COMPANY_TYPE_SYSTEM.equals(operatorCompanyTypeId)) {
       return; // 允许操作
   }
   
   // 其他管理员只能操作本公司类型
   if (!operatorCompanyTypeId.equals(targetCompanyTypeId)) {
       throw new BusinessException(ErrorCode.FORBIDDEN, "无权限");
   }
   ```

## 📊 数据库设计

### users 表
- `company_id`: 用户所属公司 ID
- `approval_status`: 审批状态（0:待审批 1:通过 2:拒绝）
- `approved_by`: 审批人 ID
- `status`: 用户状态（0:待审批 1:正常 2:禁用）

### companies 表
- `type_id`: 公司类型 ID（1:甲方 2:乙方 3:监理方 4:系统管理员）

### company_types 表
- `id`: 公司类型 ID
- `type_name`: 公司类型名称

## 🔒 安全考虑

1. **后端权限校验**: 所有敏感操作都在后端进行权限校验
2. **防止越权**: 通过公司类型隔离不同管理员的权限范围
3. **系统保护**: 系统管理员拥有最高权限，可以管理所有资源
4. **审计日志**: 所有操作都记录操作人 ID，便于审计
