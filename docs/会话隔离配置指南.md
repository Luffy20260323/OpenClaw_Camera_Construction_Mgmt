## 核心概念

### 会话隔离是什么？

会话隔离决定：**不同用户、不同渠道、不同群组的对话内容是否共享**。

正确的隔离配置可以确保：
- ✅ 群聊内容只在群内共享
- ✅ 私聊内容严格保密
- ✅ 不同群组之间互不干扰
- ✅ 多机器人协同工作时上下文对齐

## 配置参数详解

### 第 1 层：`agents.list[]` — 划定子公司范围

**作用**：创建多个独立的 AI 代理实例

```json
{
  "agents": {
    "list": [
      { "id": "main", "default": true, "workspace": "~/.openclaw/workspace-main" },
      { "id": "dev", "workspace": "~/.openclaw/workspace-dev" },
      { "id": "test", "workspace": "~/.openclaw/workspace-test" }
    ]
  }
}
```

| 参数 | 说明 |
|------|------|
| `id` | Agent 唯一标识 |
| `default` | 是否为默认 Agent |
| `workspace` | 独立工作区路径（隔离记忆和文件） |

---

### 第 2 层：`bindings[]` — 员工分配

**作用**：将渠道账号绑定到指定 Agent

```json
{
  "bindings": [
    {
      "agentId": "dev",
      "match": {
        "channel": "feishu",
        "accountId": ["dev-1", "dev-2"]
      }
    },
    {
      "agentId": "test",
      "match": {
        "channel": "feishu",
        "accountId": ["test-1"]
      }
    }
  ]
}
```

**效果**：
- `dev-1` 和 `dev-2` 归属于 `dev` Agent
- `test-1` 归属于 `test` Agent
- 不同 Agent 之间完全隔离（独立记忆、独立工作区）

---

### 第 3 层：`session.dmScope` — 对外对话隔离粒度

**作用**：控制员工与客户的对话如何组织

```json
{
  "session": {
    "dmScope": "per-channel-peer"
  }
}
```

| 值 | 效果 | 适用场景 |
|---|------|---------|
| `main` | 所有人共享一个大聊天室 | 单人使用，多设备 |
| `per-peer` | 按客户隔离，员工间共享 | 客服团队，客户档案统一 |
| `per-channel-peer` ⭐ | 按渠道 + 客户隔离 | 多人多场景，推荐默认 |
| `per-account-channel-peer` | 按员工 + 渠道 + 客户隔离 | 完全隔离，隐私优先 |

---

### 第 4 层：`groupSessionScope` — 群聊共享策略

**作用**：控制群聊上下文是否共享

```json
{
  "channels": {
    "feishu": {
      "groupSessionScope": "group"
    }
  }
}
```

| 值 | 效果 | 适用场景 |
|---|------|---------|
| `group` ⭐ | 群内所有人共享上下文 | 多机器人协同服务群聊 |
| `group_sender` | 每个员工独立群聊记录 | 机器人独立人设 |

**配置位置**：
- 渠道级别：`channels.feishu.groupSessionScope`（推荐）
- 账号级别：`channels.feishu.accounts.dev-1.groupSessionScope`（不推荐，易冲突）

## 8 种标准隔离场景

| # | 员工范围 | 客户范围 | 配置组合 | 会话 Key 示例 |
|---|---------|---------|---------|--------------|
| 1 | 1 个员工 | 1 个客户 | `per-account-channel-peer` | `agent:main:feishu:A1:direct:C10` |
| 2 | 1 个员工 | 全部客户 | 需自定义绑定 | `agent:A1:main` |
| 3 | 1 部门 | 1 个客户 | `per-channel-peer` | `agent:main:feishu:direct:C10` |
| 4 | 1 部门 | 全部客户 | `dmScope: main` + 绑定 | `agent:D1:main` |
| 5 | 1 子公司 | 1 个客户 | 多 Agent + `per-channel-peer` | `agent:S1:feishu:direct:C10` |
| 6 | 1 子公司 | 全部客户 | 多 Agent + `dmScope: main` | `agent:S1:main` |
| 7 | 1 集团 | 1 个客户 | `per-peer` | `agent:main:direct:C10` |
| 8 | 1 集团 | 全部客户 | `dmScope: main` | `agent:main:main` |

## 实战案例

### 案例 1：多机器人协同服务群聊 ⭐ 推荐

**需求**：
- 5 个机器人 + 2 个用户在同一个飞书群
- 群内所有人共享会话上下文
- 不同群之间隔离
- 私聊独立

**配置**：
```json
{
  "session": {
    "dmScope": "per-channel-peer"
  },
  "channels": {
    "feishu": {
      "groupSessionScope": "group"
    }
  }
}
```

**效果验证**：

| 场景 | 会话 Key | 是否共享 |
|------|---------|---------|
| 群 1：用户 1 发消息 | `agent:main:feishu:group:群 1ID` | ✅ 群 1 内共享 |
| 群 1：机器人 3 回复 | `agent:main:feishu:group:群 1ID` | ✅ 同上 |
| 群 2：用户 1 发消息 | `agent:main:feishu:group:群 2ID` | ❌ 群间隔离 |
| 私聊：用户 1↔机器人 1 | `agent:main:feishu:direct:用户 1` | ❌ 私聊独立 |

---

### 案例 2：客服团队（客户档案统一）

**需求**：
- 多个客服账号
- 同一个客户找任何客服看到同一历史
- 不同客户之间隔离

**配置**：
```json
{
  "session": {
    "dmScope": "per-peer",
    "identityLinks": {
      "客户 A": ["feishu:客户 A-id", "whatsapp:客户 A-phone"]
    }
  }
}
```

**效果**：
- ✅ 客户 A 找客服 1 或客服 2，看到同一档案
- ❌ 客户 B 看不到客户 A 的内容

---

### 案例 3：多子公司完全隔离

**需求**：
- 开发团队和测试团队完全隔离
- 各自有独立的工作区和记忆

**配置**：
```json
{
  "agents": {
    "list": [
      { "id": "dev", "workspace": "~/.openclaw/workspace-dev" },
      { "id": "test", "workspace": "~/.openclaw/workspace-test" }
    ]
  },
  "bindings": [
    { "agentId": "dev", "match": { "channel": "feishu", "accountId": ["dev-1", "dev-2"] } },
    { "agentId": "test", "match": { "channel": "feishu", "accountId": ["test-1"] } }
  ],
  "session": {
    "dmScope": "per-channel-peer"
  }
}
```

**效果**：
- ✅ 开发团队和测试团队完全隔离
- ✅ 各自有独立的记忆和文件

## 配置检查清单

### 部署前检查

- [ ] 确定需要几个 Agent（子公司）
- [ ] 确定每个 Agent 管理哪些账号（bindings）
- [ ] 确定 dmScope 隔离粒度
- [ ] 确定 groupSessionScope 共享策略
- [ ] 检查是否有账号级别配置冲突

### 验证步骤

1. **群聊共享验证**
   ```
   步骤：
   1. 在群里@机器人 A 问问题
   2. 在群里@机器人 B 问"刚才聊了什么"
   3. 机器人 B 应该能看到上下文
   ```

2. **群间隔离验证**
   ```
   步骤：
   1. 在群 1 里讨论话题 A
   2. 在群 2 里@机器人问"刚才聊了什么"
   3. 机器人应该看不到群 1 的内容
   ```

3. **私聊隔离验证**
   ```
   步骤：
   1. 用户 A 私聊机器人
   2. 用户 B 私聊机器人
   3. 两个对话应该完全独立
   ```

## 常见问题

### Q1: 配置修改后不生效？

**A**: 需要重启 Gateway：
```bash
openclaw gateway restart
```

### Q2: 账号级别和渠道级别配置冲突？

**A**: 账号级别配置优先级更高。建议：
- 删除所有账号级别的 `groupSessionScope`
- 只保留渠道级别的统一配置

### Q3: 如何查看当前会话 Key？

**A**: 发送 `/status` 命令查看当前会话信息。

### Q4: 网页聊天和飞书聊天会共享吗？

**A**: 默认不会。`dmScope: "per-channel-peer"` 会按渠道隔离。
如需共享，改为 `dmScope: "per-peer"`。

### Q5: 多个 Gateway 如何配置？

**A**: 每个 Gateway 独立配置，互不影响。如需同步状态，需要外部协调。

## 附录：完整配置示例

```json5
// ~/.openclaw/openclaw.json
{
  "agents": {
    "defaults": {
      "model": { "primary": "bailian/qwen3.5-plus" }
    }
  },
  
  "session": {
    "dmScope": "per-channel-peer"  // 按渠道 + 用户隔离
  },
  
  "channels": {
    "feishu": {
      "enabled": true,
      "connectionMode": "websocket",
      "groupSessionScope": "group",  // 群内共享
      "accounts": {
        "dev-1": {
          "name": "开发 1",
          "appId": "cli_a93e6ea211b89cd2",
          "appSecret": "xxx",
          "enabled": true
          // 不要在这里配置 groupSessionScope，避免冲突
        },
        "dev-2": {
          "name": "开发 2",
          "appId": "cli_a93e6f4f2b399cc8",
          "appSecret": "xxx",
          "enabled": true
        }
      }
    }
  }
}
```
