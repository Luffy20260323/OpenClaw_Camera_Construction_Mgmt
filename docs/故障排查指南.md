## 1. 快速诊断流程

### 1.1 5 分钟快速检查

```bash
# 1. 检查所有服务状态
systemctl status nginx postgresql redis-server

# 2. 检查后端进程
pgrep -f "java.*jar" && echo "✅ 后端运行中" || echo "❌ 后端未运行"

# 3. 检查端口监听
ss -tlnp | grep -E "(80|443|8080|5432|6379)"

# 4. 测试 API 连通性
curl -I http://localhost:8080/api/health

# 5. 测试前端访问
curl -I http://localhost/
```

### 1.2 问题定位决策树

```
用户报告问题
    │
    ├── 页面完全无法访问 (502/503/连接超时)
    │   ├── 检查 Nginx 状态 → systemctl status nginx
    │   ├── 检查后端状态 → pgrep -f java
    │   └── 检查端口监听 → ss -tlnp | grep :80
    │
    ├── 页面能打开但功能异常
    │   ├── 打开浏览器控制台 (F12) 查看错误
    │   ├── 检查后端日志 → tail -f /var/log/camera-backend.log
    │   └── 检查 API 响应 → curl http://localhost:8080/api/xxx
    │
    ├── 页面布局错乱/样式丢失
    │   ├── 检查前端文件 → ls -la /var/www/html/assets/
    │   ├── 清除浏览器缓存
    │   └── 重新构建前端 → cd frontend && npm run build
    │
    └── 登录失败/认证错误
        ├── 检查 Redis → redis-cli ping
        ├── 检查数据库 → psql -U postgres -c "SELECT 1"
        └── 查看认证日志 → grep -i "auth" /var/log/camera-backend.log
```

## 4. 前端问题排查

### 4.1 页面完全空白

**现象**: 浏览器显示空白页面，控制台有错误

**排查步骤**:

```bash
# 1. 检查前端文件是否存在
ls -la /var/www/html/
ls -la /var/www/html/assets/

# 2. 检查 Nginx 配置
cat /etc/nginx/sites-available/camera-system

# 3. 测试 Nginx 配置
nginx -t

# 4. 查看 Nginx 错误日志
tail -100 /var/log/nginx/error.log

# 5. 查看 Docker 容器日志
docker-compose logs nginx
```

**常见原因**:
- 前端文件未构建或构建失败
- Nginx 配置错误
- 浏览器缓存问题

**解决方案**:

```bash
# 重新构建前端
cd /root/.openclaw/workspace/OpenClaw_Camera_Construction_Mgmt/frontend
npm install
npm run build

# 如果使用 Docker，重新构建容器
docker-compose up -d --build frontend

# 清除浏览器缓存 (用户侧操作)
# Ctrl+Shift+Delete 或 无痕模式访问
```

### 4.2 页面布局错乱/样式丢失

**现象**: 页面能打开，但样式不正确，按钮/表格显示异常

**排查步骤**:

```bash
# 1. 打开浏览器开发者工具 (F12)
# 2. 查看 Console 标签页的错误
# 3. 查看 Network 标签页，检查 CSS/JS 文件是否加载成功 (状态码 200)

# 4. 服务器端检查
ls -lh /var/www/html/assets/
# 检查文件大小，如果为 0 或异常小，说明构建有问题
```

**解决方案**:

```bash
# 重新构建前端
cd /root/.openclaw/workspace/OpenClaw_Camera_Construction_Mgmt/frontend

# 清理缓存
rm -rf node_modules/.vite
rm -rf dist/

# 重新安装依赖并构建
npm install
npm run build

# 重新加载 Nginx
docker-compose restart nginx
# 或
nginx -s reload
```

### 4.3 功能按钮点击无响应

**现象**: 页面正常，但点击按钮后无反应

**排查步骤**:

```bash
# 1. 打开浏览器开发者工具 (F12)
# 2. 查看 Console 是否有 JavaScript 错误
# 3. 查看 Network 标签页，检查 API 请求状态

# 4. 检查后端 API 是否正常
curl -X GET http://localhost:8080/api/health

# 5. 检查后端日志
docker-compose logs -f backend
# 或
tail -f /var/log/camera-backend.log
```

**常见原因**:
- JavaScript 错误阻止了事件处理
- API 请求失败 (401/403/500)
- 网络请求被 CORS 策略阻止

### 4.4 登录页面问题

**现象**: 无法登录，提示各种错误

**排查清单**:

| 错误提示 | 可能原因 | 检查命令 |
|---------|---------|---------|
| 用户名或密码错误 | 密码错误/用户不存在 | `psql -U postgres -c "SELECT username FROM users WHERE username='xxx';"` |
| 验证码错误 | Redis 服务异常 | `redis-cli ping` |
| 账号未审批 | 用户状态为待审批 | `psql -U postgres -c "SELECT approval_status FROM users WHERE username='xxx';"` |
| 账号已禁用 | 用户状态为禁用 | `psql -U postgres -c "SELECT status FROM users WHERE username='xxx';"` |
| 网络错误 | 后端服务不可用 | `curl http://localhost:8080/api/auth/login` |

## 6. 数据库问题排查

### 6.1 数据库连接检查

```bash
# 方式 1: Docker 容器内
docker-compose exec postgres psql -U postgres

# 方式 2: 直接连接 (如果暴露了端口)
psql -h localhost -U postgres -d camera_construction_db

# 方式 3: 检查连接
docker-compose exec postgres pg_isready
```

### 6.2 数据库状态检查

```sql
-- 连接到数据库
\c camera_construction_db

-- 查看表列表
\dt

-- 查看用户表数据
SELECT id, username, email, status, approval_status, created_at 
FROM users 
ORDER BY created_at DESC 
LIMIT 10;

-- 查看连接数
SELECT count(*) FROM pg_stat_activity;

-- 查看数据库大小
SELECT pg_size_pretty(pg_database_size('camera_construction_db'));

-- 查看锁状态
SELECT * FROM pg_locks WHERE granted = false;

-- 查看慢查询
SELECT query, calls, total_time, mean_time 
FROM pg_stat_statements 
ORDER BY total_time DESC 
LIMIT 10;
```

### 6.3 数据库备份与恢复

```bash
# 备份数据库
docker-compose exec postgres pg_dump -U postgres camera_construction_db > backup_$(date +%Y%m%d_%H%M%S).sql

# 压缩备份
docker-compose exec postgres pg_dump -U postgres camera_construction_db | gzip > backup_$(date +%Y%m%d_%H%M%S).sql.gz

# 恢复数据库
docker-compose exec -T postgres psql -U postgres camera_construction_db < backup_20260326_120000.sql

# 或从压缩文件恢复
gunzip -c backup_20260326_120000.sql.gz | docker-compose exec -T postgres psql -U postgres camera_construction_db
```

### 6.4 数据库优化

```sql
-- 清理空闲空间
VACUUM ANALYZE;

-- 清理特定表
VACUUM ANALYZE users;

-- 重建索引
REINDEX TABLE users;

-- 查看表大小
SELECT 
    tablename,
    pg_size_pretty(pg_total_relation_size(schemaname||'.'||tablename)) AS size
FROM pg_tables
WHERE schemaname = 'public'
ORDER BY pg_total_relation_size(schemaname||'.'||tablename) DESC;
```

```nginx
# 确保以下配置正确
server {
    listen 80;
    server_name _;
    
    # 前端静态文件
    location / {
        root /var/www/html;
        try_files $uri $uri/ /index.html;
    }
    
    # API 反向代理
    location /api/ {
        proxy_pass http://backend:8080/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

## 9. 常见问题速查

### 9.1 快速修复命令

```bash
# === 服务重启 ===
# 重启所有服务
docker-compose restart

# 重启单个服务
docker-compose restart nginx
docker-compose restart backend
docker-compose restart postgres
docker-compose restart redis

# === 日志查看 ===
# 实时查看日志
docker-compose logs -f

# 查看特定服务日志
docker-compose logs -f backend
docker-compose logs -f nginx

# 查看最近 100 行
docker-compose logs --tail=100 backend

# === 健康检查 ===
# 检查所有容器
docker-compose ps

# 检查 API
curl http://localhost:8080/api/health

# 检查前端
curl -I http://localhost/

# 检查数据库
docker-compose exec postgres pg_isready

# 检查 Redis
docker-compose exec redis redis-cli ping
```

### 9.2 问题 - 解决方案对照表

| 问题 | 检查点 | 解决方案 |
|------|--------|---------|
| 502 Bad Gateway | 后端服务 | `docker-compose restart backend` |
| 503 Service Unavailable | Nginx/后端 | `docker-compose restart nginx backend` |
| 页面空白 | 前端文件/浏览器缓存 | 重新构建前端，清除缓存 |
| 样式丢失 | assets 目录 | `cd frontend && npm run build` |
| 登录失败 | Redis/数据库 | `docker-compose restart redis postgres` |
| API 超时 | 后端性能/数据库 | 查看后端日志，优化数据库 |
| 文件上传失败 | uploads 目录权限 | `chmod 755 uploads/` |
| 验证码不显示 | Redis | `docker-compose restart redis` |

### 9.3 应急恢复流程

```bash
# 1. 备份当前状态
docker-compose ps > /tmp/container-status.txt
docker-compose logs > /tmp/container-logs.txt

# 2. 尝试重启
docker-compose restart

# 3. 如果不行，重建容器
docker-compose down
docker-compose up -d

# 4. 如果还不行，重新构建
docker-compose down
docker-compose up -d --build

# 5. 最后手段：完全重置 (会丢失数据!)
# docker-compose down -v
# docker-compose up -d --build
```
