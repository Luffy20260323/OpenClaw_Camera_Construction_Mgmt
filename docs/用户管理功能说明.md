# 用户管理功能实现说明

## 功能概述

实现了完整的用户管理体系，支持三种注册方式和分级审批流程。

## 三种注册方式

### 1. 用户自主注册（需审批）
- **接口**: `POST /user/register`
- **流程**: 用户填写注册信息 → 提交申请 → 等待管理员审批 → 审批通过后才能登录
- **审批规则**: 
  - 乙方公司用户 → 乙方管理员或系统管理员审批
  - 甲方公司用户 → 甲方管理员或系统管理员审批
  - 系统管理员 → 可审批所有用户

### 2. 管理员手动创建
- **接口**: `POST /user`
- **权限**: 
  - 系统管理员：可创建任何用户
  - 甲方管理员：只能创建甲方公司用户
  - 乙方管理员：只能创建乙方公司用户
  - 监理方管理员：只能创建监理方公司用户
- **参数**: `autoApprove` 可选，默认为 false（需要审批），设置为 true 则直接通过

### 3. 管理员批量导入
- **接口**: `POST /user/batch-import`
- **格式**: JSON 数组，支持多条用户记录
- **权限**: 同手动创建
- **参数**: `autoApprove` 可选，控制是否直接通过审批

## 核心 API

### 用户注册
```http
POST /user/register
Content-Type: application/json

{
  "username": "testuser",
  "password": "123456",
  "confirmPassword": "123456",
  "realName": "测试用户",
  "email": "test@example.com",
  "phone": "13800138000",
  "companyId": 1,
  "gender": 1
}
```

### 管理员创建用户
```http
POST /user
Authorization: Bearer <token>
Content-Type: application/json

{
  "username": "newuser",
  "password": "123456",
  "confirmPassword": "123456",
  "realName": "新用户",
  "email": "newuser@example.com",
  "phone": "13800138001",
  "companyId": 1,
  "roleIds": [1, 2],
  "gender": 1,
  "autoApprove": true
}
```

### 批量导入用户
```http
POST /user/batch-import
Authorization: Bearer <token>
Content-Type: application/json

{
  "autoApprove": true,
  "users": [
    {
      "username": "user1",
      "password": "123456",
      "realName": "用户 1",
      "email": "user1@example.com",
      "companyId": 1,
      "roleIds": [1]
    },
    {
      "username": "user2",
      "password": "123456",
      "realName": "用户 2",
      "email": "user2@example.com",
      "companyId": 1,
      "roleIds": [2]
    }
  ]
}
```

### 审批用户
```http
POST /user/approve
Authorization: Bearer <token>
Content-Type: application/json

{
  "userId": 123,
  "approved": true
}
```

拒绝审批：
```json
{
  "userId": 123,
  "approved": false,
  "rejectionReason": "信息不完整"
}
```

### 查询待审批用户
```http
GET /user/pending-approvals
Authorization: Bearer <token>
```

### 分页查询用户列表
```http
GET /user/list?pageNum=1&pageSize=10&companyTypeId=1&approvalStatus=0
Authorization: Bearer <token>
```

查询参数：
- `pageNum`: 页码（默认 1）
- `pageSize`: 每页数量（默认 10）
- `username`: 用户名（模糊搜索）
- `realName`: 真实姓名（模糊搜索）
- `companyId`: 公司 ID
- `companyTypeId`: 公司类型 ID
- `roleId`: 角色 ID
- `approvalStatus`: 审批状态（0:待审批 1:通过 2:拒绝）
- `status`: 用户状态（0:待审批 1:正常 2:禁用）
- `keyword`: 关键词（搜索用户名、姓名、手机、邮箱）

Content-Type: application/json

{
  "username": "yifang_user",
  "password": "123456",
  "confirmPassword": "123456",
  "realName": "乙方用户",
  "email": "yifang_user@example.com",

  "roleIds": [<乙方普通用户角色 ID>],

### 第三步：测试自主注册流程
```http
POST /user/register
Content-Type: application/json

{
  "username": "zizhu_user",
  "password": "123456",
  "confirmPassword": "123456",
  "realName": "自主注册用户",
  "email": "zizhu@example.com",
  "companyId": <乙方公司 ID>
}
```

此时用户无法登录（审批状态为待审批）。

### 第四步：乙方管理员审批
```http
GET /user/pending-approvals
Authorization: Bearer <yifang_admin_token>

POST /user/approve
Authorization: Bearer <yifang_admin_token>
Content-Type: application/json

{
  "userId": <刚注册的用户 ID>,
  "approved": true
}
```

审批通过后，用户可以正常登录。

## 权限控制逻辑

### 管理员创建用户时的权限校验
1. **系统管理员**: `companyTypeId` 为系统管理员类型，可创建任何公司的用户
2. **甲方管理员**: 只能创建 `companyTypeId = 1`（甲方）的用户
3. **乙方管理员**: 只能创建 `companyTypeId = 2`（乙方）的用户
4. **监理方管理员**: 只能创建 `companyTypeId = 3`（监理方）的用户

### 审批权限校验
1. 获取审批人的公司类型
2. 只能审批同公司类型的待审批用户
3. 系统管理员可审批所有用户

## 数据库字段说明

### users 表
- `approval_status`: 审批状态（0:待审批 1:通过 2:拒绝）
- `approved_by`: 审批人 ID
- `approved_at`: 审批时间
- `rejection_reason`: 拒绝原因
- `status`: 用户状态（0:待审批 1:正常 2:禁用）

## 错误码

- `USER_NOT_APPROVED(2004)`: 用户尚未审批通过（登录时返回）
- `USERNAME_EXISTS(2005)`: 用户名已存在
- `EMAIL_EXISTS(2006)`: 邮箱已被注册
- `PHONE_EXISTS(2007)`: 手机号已被注册

## 注意事项

1. **密码安全**: 所有密码都经过 BCrypt 加密存储
2. **事务控制**: 创建用户和分配角色在同一事务中，保证数据一致性
3. **批量导入**: 支持部分成功，失败的用户会返回错误信息
4. **审批流程**: 自主注册的用户必须经过审批才能登录
5. **权限隔离**: 管理员只能管理自己公司范围内的用户
