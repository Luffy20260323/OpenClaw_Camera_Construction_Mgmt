# 用户注册 - 公司角色联动前端测试用例

**版本**: v5.0-Frontend  
**日期**: 2026-03-21  
**类型**: 前端手动测试  
**优先级**: P0  

---

## 📋 测试场景

用户在登录页面进行注册登记时，公司类型、角色、作业区三者之间的联动逻辑：

1. **选择公司后** → 角色列表联动更新为该公司类型的角色
2. **选择作业区角色后** → 显示作业区选择框（仅甲方作业区角色）
3. **选择非作业区角色后** → 不显示作业区选择框

---

| **前置条件** | 系统有多种角色 |
| **测试步骤** | 1. 查看系统角色列表<br>2. 识别哪些角色名称包含"作业区"<br>3. 逐一选择这些角色，观察作业区选择框 |
| **预期结果** | 1. 角色名称包含"作业区"的角色（如"作业区项目主管"、"作业区项目经理"、"作业区项目经办人"）→ **显示**作业区选择框<br>2. 角色名称不包含"作业区"的角色 → **不显示**作业区选择框<br>3. 判断逻辑一致，无例外 |
